# Программный комплекс «Фурлаб»

**Программный комплекс «Фурлаб»** предназначен для автоматизации учёта и рационального использования лоскутных остатков тканей в условиях мелкосерийного швейного производства. Комплекс обеспечивает полный цикл работы с вторичными материалами: от регистрации поступления лоскута до формирования отчёта об использовании при раскрое.

## Функциональные задачи

- Ведение инвентарного учёта лоскутных остатков с атрибутами материала, размерами и QR-идентификацией
- Визуализация и ручное размещение фрагментов лекал на лоскутах
- Расчёт показателей использования материала: площадь покрытия, припуск на швы, остаток
- Сохранение и восстановление состояния выкладки в рамках проекта
- Формирование отчётов по размещениям и история операций

## Входные данные

Программный комплекс принимает файлы лекал в формате **DXF** — открытом векторном формате, поддерживаемом большинством CAD/CAM-систем швейной отрасли. Это обеспечивает независимость от конкретного ПО проектирования: файлы могут быть получены из любой совместимой системы (Grafis, Lectra, Optitex и др.).

В качестве рекомендуемой среды подготовки данных выбрана **CLO 3D** — на момент начала разработки единственное решение, удовлетворявшее требованиям работы с аватарами, трёхмерными примерками и фотореалистичным рендерингом материалов, включая мех.

## Состав программного комплекса

| Модуль | Назначение |
|--------|-----------|
| **Веб-приложение учёта** (`furlab-access`) | Инвентаризация лоскута, история размещений, справочники |
| **Сервис выкладки** (`furlab-web-plugin`) | Визуализация зон и фрагментов, ручная выкладка, отчёты |
| **Сервер приложений** (Node.js) | REST API, бизнес-логика, межсервисное взаимодействие |
| **База данных** (Microsoft Access `.accdb`) | Каталог материалов, инвентарные записи, параметры выкладки |
| **Журнал трассировки** (NDJSON) | Append-лог размещений для аудита и истории операций |

## Архитектура взаимодействия

```
CLO 3D / любая CAD-система
  └── экспорт DXF (геометрия лекал)
        ↓
furlab-web-plugin  (сервис выкладки)
  ├── React UI :5173
  └── Node.js API :5600
        └── /api/ac-proxy/* ──→ furlab-access :5500

furlab-access  (сервис учёта)
  ├── React UI :5173/furlab-ac
  └── Node.js API :5500
        ├── Access DB (.accdb)
        └── NDJSON-журнал
```
