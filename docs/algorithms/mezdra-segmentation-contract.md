# Короткий контракт по V2 и V3

## 1. Общая рамка

- **V1** зафиксирован как legacy baseline и в развитие не входит.
- **V2** — безопасный `refine-only` pipeline.
- **V3** — альтернативная логика сегментации маски мездры.

Правило сравнения:
- **качество рабочей сегментации сравнивается только между `V2` и `V3`**;
- **`V1` используется только как legacy/demo reference** и для проверки, что legacy-режим не сломан.

Цель этапа: получить более корректный контур мездры без меховой бахромы, не ломая UI и не ухудшая legacy-кейсы.

---

## 2. Разделение V2 и V3

### V2
V2 работает только как постобработка уже найденной маски.

Допускается:
- легкая морфология;
- удаление мелких выбросов;
- сглаживание границы;
- заполнение мелких выемок;
- fallback на исходную маску.

Не допускается:
- менять базовый способ первичного выделения маски;
- ML;
- тяжелые операции, вызывающие фриз UI.

### V3
V3 может менять способ построения самой маски.

Допускается:
- иные пороги и правила сегментации;
- дополнительные признаки;
- alternative rule-based logic;
- собственный refine, если соблюдены лимиты времени и fallback.

Не допускается:
- ML в рамках этого этапа;
- обязательная ручная настройка под каждый кадр;
- отсутствие fallback.

---

## 3. Общие рамки реализации

В рамках этапа можно менять:
- получение маски;
- refinement/post-process;
- параметры pipeline;
- debug-режим;
- переключение версий в UI.

Не входят в этап:
- переработка QR-детекции;
- переработка логики линии/направления ворса;
- прочая инфраструктура вне сегментации.

---

## 4. Общий результат pipeline

Каждая версия V2 и V3 должна возвращать:
- `finalMask`
- `bbox`
- `area`
- `processingTimeMs`
- `refineApplied`
- `fallbackUsed`
- `timeoutHit`

`finalMask`:
- размер как у исходного изображения;
- бинарная маска `0/255`;
- координаты: `x` вправо, `y` вниз.

Если есть contour, он должен быть:
- внешним;
- замкнутым;
- по основной компоненте.

---

## 5. Ограничения

Обязательно:
- без фриза UI;
- детерминированный результат на одинаковом входе;
- fallback при ошибке или таймауте;
- одна основная компонента в результате.

Если компонент несколько:
- берется крупнейшая;
- если вторая слишком крупная, кейс помечается как неоднозначный.

---

## 6. Таймауты

- `softTimeoutMs = 180 ms`
- `hardTimeoutMs = 600 ms`

При `soft timeout`:
- останавливаются необязательные этапы;
- pipeline пытается завершиться упрощенно.

При `hard timeout`:
- pipeline прекращается;
- возвращается безопасный fallback-результат;
- ставятся флаги `timeoutHit = true`, `fallbackUsed = true`.

---

## 7. Набор для проверки

Минимальный test set:
- `4` legacy-скана;
- `8` реальных сканов;
- `4` сложных скана.

Желательно:
- небольшой golden set с ручной разметкой.

---

## 8. Критерии приемки

### Для V2
V2 принимается, если:
- не фризит UI;
- не ломает legacy-кейсы;
- на реальных сканах уменьшает рваность и захват ворса;
- укладывается в лимиты времени.

### Для V3
V3 принимается, если:
- дает более корректный контур мездры, чем V2;
- не требует ручной подстройки на каждый кадр;
- укладывается в лимиты времени;
- не нарушает legacy-допуски.

---

## 9. Автотесты

Обязательные автотесты:

### 9.1. Smoke tests
- pipeline возвращает непустой результат;
- `finalMask` имеет правильный размер;
- `bbox` валиден;
- `area > 0`.

### 9.2. Determinism tests
- одинаковый вход + одинаковая версия => одинаковый `finalMask`.

### 9.3. Timeout / fallback tests
- при искусственном timeout выставляются `timeoutHit` и `fallbackUsed`;
- pipeline не падает и возвращает валидный результат.

### 9.4. Component tests
- если компонент несколько, выбирается крупнейшая;
- неоднозначный кейс корректно помечается флагом.

### 9.5. Legacy regression tests
- для legacy-набора:
  - `abs(deltaAreaPct) <= 3%`
  - `abs(deltaBBoxWPct) <= 2%`
  - `abs(deltaBBoxHPct) <= 2%`

### 9.6. Performance tests
Для изображений 2–4 MP:
- `p95(processingTimeMs) <= 250 ms`
- `max(processingTimeMs) <= 600 ms`

---

## 10. Правило сравнения V2 и V3

V3 считается лучше V2, если:
- на реальных сканах контур мездры визуально корректнее;
- и при наличии golden set есть улучшение boundary-метрик;
- и при этом V3 не выходит за лимиты времени и не ломает legacy-набор.

Важно:
- `V1` не участвует в сравнении качества на реальных сканах;
- проверки `V1` ограничены регрессией legacy-демо сценария.

---

## 11. Итог

- **V2** — улучшение уже найденной маски.
- **V3** — альтернативное построение маски.
- Если V2 и V3 не дают нужной стабильности, следующий шаг — обсуждение V4.
