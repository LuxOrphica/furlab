# Алгоритм детекции направления ворса

Последнее обновление: 2026-05-23  
Реализация: `src/legacy/appCore.js`, `src/legacy/lineDetectionHeuristics.js`

---

## 1. Постановка задачи

На скане шкурки мездрой вверх оператор наносит чёрный маркер — короткий отрезок, ориентированный вдоль направления ворса. Конец отрезка, ближайший к краю мездры, указывает направление роста волоса.

Задача системы:
1. Найти этот отрезок автоматически.
2. Определить его ориентацию как вектор (угол в градусах).
3. Установить направление вектора: от центра к ближайшему краю.

---

## 2. Общая схема pipeline

```
Исходный растр (source canvas)
    │
    ▼
Маска полигона (polygonMask) + карта расстояний до края (edgeDistanceMap)
    │
    ▼
Зона исключения QR-наклейки (stickerBox)
    │
    ▼
[Детектор 1] dark-line (компонентный анализ)  ←── ПЕРВИЧНЫЙ
    │
    ├── найдено → PCA по лучшему компоненту → сегмент
    │
    └── не найдено
         │
         ▼
    [Детектор 2] opencv-hough (HoughLinesP)  ←── РЕЗЕРВНЫЙ
         │
         └── найдено → лучший отрезок по scoring
    │
    ▼
isAutoSegmentPlausible (фильтр правдоподобия)
    │
    ▼
computeNapVectorFromLine (определение направления)
    │
    ▼
Вектор + угол ворса
```

## 3. Определение зоны исключения QR (stickerBox)

QR-наклейка — белый прямоугольный стикер с чёрными модулями QR-кода. Её нужно исключить из поиска, иначе тёмные модули QR будут восприниматься как нап-полоска.

**Проблема:** модули QR-кода разбивают белую наклейку на десятки изолированных белых островков. Простой BFS по ярким пикселям не собирает их в один компонент.

**Решение:**
1. Выделить все пиксели с яркостью > 230 внутри polygonMask (edgeDistance > 5px).
2. Применить морфологическую дилатацию радиуса 2px — соединяет острова через узкие чёрные полосы QR.
3. Обрезать результат обратно по polygonMask (иначе захватывается светлый фон сканера).
4. BFS по дилатированной маске → найти крупнейший связный компонент.
5. Его bbox + отступ 16px = зона исключения.

Пиксели внутри зоны исключения не участвуют в детекции линии на всех этапах.

## 4. Детектор 1: dark-line (компонентный анализ)

### 4.1 Бинаризация по адаптивному порогу

Вычисляется по внутренней области полигона (edgeDistance > 4px, без stickerExclusion):

```
mean = среднее значение серого по ROI
std  = стандартное отклонение
thr  = clamp(mean - 1.0 * std, 70, 185)
```

Пиксели с `gray < thr AND edgeDistance > 2.5px` попадают в rawMask.

После бинаризации — морфологическое закрытие (dilate→erode, радиус 1) для соединения разрывов тонкого штриха → `lineMaskClean`.

### 4.2 Компонентный анализ

BFS по `lineMaskClean`, ограничения:
- размер компонента: от 4 до 15 000 пикселей
- `bbox(W, H) <= 600px` (исключает крупные тёмные области меха)
- **`meanGray < 85`** — компонент должен быть реально чёрным, не тёмно-коричневым мехом или пятнами мездры

> **Обоснование meanGray < 85:** нап-полоска рисуется чёрным маркером (яркость 10–60). Тёмно-коричневый мех и тени на мездре имеют яркость 90–140. Абсолютный порог надёжнее локального контраста для данного класса объектов.

Для каждого компонента вычисляются признаки (через PCA):
- `lengthPx` — длина главной оси
- `ratio` = lengthPx / shortAxis (вытянутость)
- `linearity` — линейность формы
- `fillRatio` — заполнение bbox
- `meanEdgeDist` — среднее расстояние до края полигона

Компонент отбраковывается если:
- midpoint или любой из концов попадает в stickerExclusion (pad=16px)
- `nearEdgeRatio` слишком высок (компонент у края меха)

### 4.3 Scoring и выбор лучшего компонента

```
score = lengthPx * 2.8 + linearity * 5.5 + ratio * 7.0
        + meanEdgeDist * 1.3 - fillRatio * 14.0
```

Приоритеты отбора: primaryPass → relaxedPass → emergency → ultimate.

`primaryPass`: ratio ≥ 1.35, linearity ≥ 2.2, lengthPx ≥ 12.

### 4.4 PCA по лучшему компоненту

После выбора best component его пиксели (lineMaskBest) прогоняются через PCA → конечные точки отрезка P1, P2.

> **Важно:** используется `lineMaskBest` (пиксели только лучшего компонента), а не `lineMaskClean` (все тёмные пиксели). Иначе PCA выдаёт главную ось всего силуэта шкурки.

## 5. Детектор 2: opencv-hough (резервный)

Используется только если dark-line не нашёл подходящего кандидата.

**Шаги:**
1. Grayscale → threshold(150, BINARY_INV) → маскирование по polygonMask без stickerExclusion.
2. Морфологическое открытие + закрытие (3×3 ядро).
3. `HoughLinesP(rho=1, theta=π/180, threshold=18, minLen=14, maxGap=8)`.
4. Для каждого отрезка: вычислить darkRatio (доля пикселей с gray < 145) вдоль отрезка.
5. Отбраковать если darkRatio < 0.65.
6. Scoring: `log1p(len) * 10 + edgeD * 6.0 + darkRatio * 80` — предпочитает короткие тёмные внутренние отрезки перед длинными краевыми.

---

## 6. Фильтр правдоподобия (isAutoSegmentPlausible)

Применяется ко всем кандидатам перед принятием:

| Условие | Значение |
|---------|---------|
| Длина отрезка | ≥ 14px |
| d1, d2, dm | конечные (не Infinity) |
| min(d1, d2) | ≥ 2.5px от края |
| avg(d1, d2, dm) | ≥ 4.0px |
| darkRatio вдоль (только hough) | ≥ 0.70 |

## 7. Определение направления вектора

```javascript
d1Px = getEdgeDistance(P1)   // расстояние P1 до края полигона
d2Px = getEdgeDistance(P2)   // расстояние P2 до края полигона

shouldSwap = (d1Px < d2Px)   // P1 ближе к краю → P1 должен быть наконечником
from = shouldSwap ? P2 : P1
to   = shouldSwap ? P1 : P2
```

**Физический смысл:** ворс растёт от центра к краю шкурки. Конец полоски, ближайший к краю мездры — это конец в направлении роста. Стрелка указывает туда.

Итоговый угол: `atan2(to.y - from.y, to.x - from.x)` в диапазоне [0°, 360°).

## 8. Диагностика в UI

В панели «Тех информация» отображается:
- `Детектор` — использованный детектор (`dark-line` / `opencv-hough`)
- `Источник линии` — `global-mask-pca` / `dark-line-fallback` / `opencv-hough`
- `P1`, `P2` — координаты в пикселях
- `d1`, `d2` — расстояния до края в мм
- `Вектор P1>P2` — нормализованный вектор
- Статистика отбраковки: `compsTotal`, `compsSticker`, `compsTooBig`, `selectedBy`

---

## 9. Ручной режим

При переключении в «Ручной выбор» оператор кликает два раза: P1 (хвост) → P2 (наконечник). Логика направления по краю не применяется — направление задаётся явно.

---

## 10. Известные ограничения

| Ситуация | Поведение |
|----------|-----------|
| Полоска горизонтальна, d1 ≈ d2 | Направление стрелки условное (разница < 1мм) |
| Кривой контур (tonal-rescue) | edgeDistanceMap неточна → возможна неверная стрелка |
| Полоска короче ~8мм | Может не пройти primaryPass; потребуется ручной режим |
| Несколько тёмных меток на мездре | Выбирается компонент с наибольшим score (обычно самый длинный) |
| QR не обнаружен (stickerBox=null) | Исключение не работает; модули QR могут быть приняты за полоску |
