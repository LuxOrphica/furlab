# Алгоритм сегментации мездры

Последнее обновление: 2026-05-23  
Реализация: `src/legacy/segmentationPipelines.js`

---

## 1. Постановка задачи

На скане шкурка лежит мездрой вверх на фоне сканера (светло-серый или белый). Задача — выделить маску именно мездры (кожаная сторона, бежево-кремового цвета), без захвата:
- меховой бахромы по краям,
- фона сканера,
- теней под шкуркой.

Выход: `polygonMask` (бинарная маска), `edgeDistanceMap` (дистанция от края).

## 3. V3 Pipeline

### 3.1 Первичная маска (buildPrimaryMaskV3)

Строится через GrabCut или цветовую сегментацию. Если первичная маска схлопывается (слишком мала или узка) — запускается tonal-rescue.

**Критерий схлопывания:**
```
count < max(3800, size * 0.05)  ИЛИ
bboxW < max(32, w * 0.38)       ИЛИ
bboxH < max(32, h * 0.38)
```

### 3.2 Tonal-rescue (fallback при схлопывании)

Применяется когда первичная маска не удалась. Ищет мездру по тональному диапазону яркости.

**Параметры:**
- `tonalLow = 118` — нижняя граница (мездра светлее тёмного меха)
- `tonalHigh = 210` — верхняя граница (ниже белого фона сканера > 220)
- Исключение: `borderMargin = max(4, round(min(w,h) * 0.01))` пикселей у краёв изображения

> **Обоснование borderMargin:** фон сканера и тени под шкуркой часто касаются границы изображения. Исключение краевых пикселей предотвращает захват этих областей в тональную маску.

После выделения тональных пикселей → BFS → крупнейший компонент = baseMask.

### 3.3 Solidify (уплотнение маски)

После получения baseMask применяется `solidifyPieceMask`:

1. **Морфологическое закрытие** с радиусом `closeRadius`:
   ```
   closeRadius = clamp(round(minDim * 0.015), 2, 8)
   ```
   При minDim=324px → closeRadius ≈ 5px.

   > **Обоснование:** волоски меха по краю создают зазоры в маске шириной 3–8px. Flood fill из фона проникает через эти зазоры и образует «дыры» внутри маски. Закрытие радиуса 5px надёжно запечатывает большинство таких зазоров без потери деталей контура.

2. **Извлечение крупнейшего компонента** — убирает изолированные куски.

3. **Flood fill внешнего фона** от границ изображения.

4. **Заполнение внутренних полостей** — всё что не reached flood fill = внутри маски.

### 3.4 Финализация

После solidify:
- `floodFillOuterBackground` → `outerBgMask`
- `buildEdgeMask` → список граничных пикселей
- `buildDistanceTransform` → `edgeDistanceMap` (расстояние каждого пикселя до ближайшего граничного)

`edgeDistanceMap` используется детектором линии ворса для определения расстояний до края мездры.

## 4. Таймауты

| Порог | Значение | Поведение |
|-------|---------|-----------|
| softTimeoutMs | 180 ms | Пропускаются необязательные этапы |
| hardTimeoutMs | 600 ms | Pipeline останавливается, возвращается безопасный результат |

При превышении `hardTimeout` → флаг `timeoutHit = true`. Это фиксируется в telemetry и отображается в «Тех информации».

**Типичное время на сканах 0.3–0.4 MP:** 600–700ms (hardTimeout часто срабатывает из-за GrabCut).

## 6. Критерии качества контура

Контур считается корректным если:
- идёт по границе мездры, не по меху
- не захватывает фон сканера и тени
- не схлопывается (bbox W и H ≥ 38% от размера изображения)
- детерминирован (одинаковый вход → одинаковый результат)

Для записи в Access используется контур `opencv-external-clean` — результат `cv.findContours` по финальной маске.

## 7. Известные ограничения

| Ситуация | Поведение |
|----------|-----------|
| GrabCut timeout (> 600ms) | Переход на tonal-rescue; контур может быть менее точным |
| Очень тёмная мездра (< 118) | Tonal-rescue не захватит; маска схлопнется |
| Мездра светлее 210 | Границы могут смешиваться с фоном сканера |
| Шкурка у края изображения | borderMargin обрезает часть маски |
| Material-aware режим | Отключён по умолчанию (флаг `window.__ldvV3MaterialAware=true`) |
