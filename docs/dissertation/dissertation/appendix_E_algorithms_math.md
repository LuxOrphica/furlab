## Е.2. Алгоритм сегментации мездры (V3)

### Е.2.1. Постановка задачи

Дано изображение $I$ размером $W \times H$. Требуется построить маску $M_{\text{mezdra}}$ — область, соответствующую кожаной стороне (мездре) шкурки, исключая фон сканера, меховую бахрому и тени.

### Е.2.2. Критерий коллапса первичной маски

Первичная маска $M_0$ считается коллапсированной если выполняется хотя бы одно из условий:

$$|M_0| < \max(3800,\ W \cdot H \cdot 0.05)$$

$$\text{bboxW}(M_0) < \max\!\left(32,\ W \cdot 0.38\right)$$

$$\text{bboxH}(M_0) < \max\!\left(32,\ H \cdot 0.38\right)$$

где $|M_0|$ — число пикселей маски, $\text{bboxW}$, $\text{bboxH}$ — ширина и высота описывающего прямоугольника.

### Е.2.3. Тональное восстановление (tonal-rescue)

При коллапсе первичной маски строится тональная маска:

$$M_{\text{tonal}}(x,y) = \begin{cases} 1, & \text{если}\ g(x,y) \in [t_l,\ t_h]\ \text{и}\ (x,y) \notin B_{\delta} \\ 0, & \text{иначе} \end{cases}$$

где:
- $t_l = 118$ — нижняя граница (мездра светлее тёмного меха),
- $t_h = 210$ — верхняя граница (ниже яркости фона сканера $> 220$),
- $B_{\delta}$ — краевая полоса шириной $\delta = \max\!\left(4,\ \lfloor\min(W,H) \cdot 0.01\rfloor\right)$ пикселей.

По $M_{\text{tonal}}$ выполняется обход в ширину (BFS); крупнейший связный компонент принимается за базовую маску $M_{\text{base}}$.

### Е.2.4. Уплотнение маски (solidify)

Для устранения зазоров, образованных волосками меха, применяется морфологическое закрытие:

$$M_{\text{solid}} = (M_{\text{base}} \oplus K_r) \ominus K_r$$

где $\oplus$ — дилатация, $\ominus$ — эрозия, $K_r$ — дисковое ядро радиуса

$$r = \text{clamp}\!\left(\left\lfloor\min(W,H) \cdot 0.015\right\rfloor,\ 2,\ 8\right).$$

После закрытия:
1. Выделяется крупнейший связный компонент $M_{\text{solid}}$.
2. От границ изображения запускается flood fill — выделяется внешний фон $M_{\text{bg}}$.
3. Итоговая маска: $M_{\text{mezdra}} = \overline{M_{\text{bg}}} \cap M_{\text{solid}}$.

### Е.2.5. Карта дистанций

Для каждого пикселя $(x,y) \in M_{\text{mezdra}}$ вычисляется расстояние до ближайшего граничного пикселя:

$$D(x,y) = \min_{(u,v) \in \partial M_{\text{mezdra}}} \sqrt{(x-u)^2 + (y-v)^2}$$

$D$ используется детектором направления ворса.

## Е.3. Алгоритм детекции направления ворса

### Е.3.1. Постановка задачи

На скане присутствует чёрная метка — отрезок, ориентированный вдоль направления роста меха. Конец отрезка, ближайший к краю мездры, указывает направление ворса. Требуется обнаружить этот отрезок и определить единичный вектор направления.

### Е.3.2. Исключение зоны QR-наклейки

QR-наклейка — белый прямоугольный стикер с тёмными модулями. Зона исключения строится следующим образом:

1. Выделить пиксели с $g(x,y) > 230$ внутри $M_{\text{mezdra}}$ с $D(x,y) > 5$:
$$M_{\text{bright}} = \{(x,y) : g(x,y) > 230,\ M_{\text{mezdra}}(x,y)=1,\ D(x,y)>5\}$$

2. Применить дилатацию радиуса 2 пикселя: $M_{\text{bright}}' = M_{\text{bright}} \oplus K_2$.

3. Ограничить маской мездры: $M_{\text{bright}}'' = M_{\text{bright}}' \cap M_{\text{mezdra}}$.

4. BFS по $M_{\text{bright}}''$ → крупнейший компонент → описывающий прямоугольник $R_{\text{QR}}$ с отступом 16 пикселей = зона исключения $Z_{\text{excl}}$.

### Е.3.3. Детектор 1: dark-line (компонентный анализ)

**Бинаризация.** По области $\{(x,y): D(x,y) > 4,\ (x,y) \notin Z_{\text{excl}}\}$ вычисляются:

$$\mu = \frac{1}{|R|}\sum_{(x,y)\in R} g(x,y), \quad \sigma = \sqrt{\frac{1}{|R|}\sum_{(x,y)\in R}(g(x,y)-\mu)^2}$$

$$t = \text{clamp}(\mu - \sigma,\ 70,\ 185)$$

Бинарная маска тёмных пикселей:

$$M_{\text{dark}}(x,y) = \begin{cases} 1, & g(x,y) < t\ \text{и}\ D(x,y) > 2.5 \\ 0, & \text{иначе} \end{cases}$$

После морфологического закрытия (радиус 1): $M_{\text{line}} = (M_{\text{dark}} \oplus K_1) \ominus K_1$.

**Фильтрация компонент.** По каждому компоненту $C_i \in \mathcal{C}(M_{\text{line}})$ вычисляются признаки через PCA:

$$\text{PCA}(C_i): \quad \lambda_1 \geq \lambda_2 \geq 0 \quad \text{(собственные значения)}$$

$$\ell_i = 2\sqrt{\lambda_1},\ \ s_i = 2\sqrt{\lambda_2}, \quad \text{ratio}_i = \frac{\ell_i}{s_i + \varepsilon}$$

$$\text{linearity}_i = \frac{\ell_i}{s_i + 1}$$

Компонент $C_i$ отбрасывается если:
- $|C_i| \notin [4,\ 15000]$,
- $\max(\text{bboxW}_i, \text{bboxH}_i) > 600$,
- $\bar{g}(C_i) > 85$ (не является чёрным маркером),
- конечные точки или центр попадают в $Z_{\text{excl}}$.

**Скоринг** для оставшихся кандидатов:

$$\text{score}_i = 2.8\,\ell_i + 5.5\,\text{linearity}_i + 7.0\,\text{ratio}_i + 1.3\,\bar{D}_i - 14.0\,\text{fill}_i$$

где $\bar{D}_i$ — среднее расстояние до края, $\text{fill}_i$ — заполнение описывающего прямоугольника.

Выбирается компонент $C^* = \arg\max_i \text{score}_i$.

**PCA по лучшему компоненту** даёт главную ось → конечные точки $P_1, P_2$.

### Е.3.4. Детектор 2: opencv-hough (резервный)

Применяется при отсутствии подходящего кандидата после работы детектора dark-line.

1. Пороговая бинаризация: $M_H = \{(x,y): g(x,y) < 150\} \cap M_{\text{mezdra}}$.
2. Морфологическое открытие + закрытие ядром $3 \times 3$.
3. Преобразование Хафа для отрезков:
$$\rho = x\cos\theta + y\sin\theta, \quad \rho \in \mathbb{R},\ \theta \in [0,\pi)$$
Параметры: $\Delta\rho=1$, $\Delta\theta=\pi/180$, порог голосования 18, минимальная длина 14, максимальный зазор 8 пикселей.
4. Для каждого отрезка $L_j$ вычисляется доля тёмных пикселей:
$$\text{darkRatio}_j = \frac{|\{p \in L_j : g(p) < 145\}|}{|L_j|}$$
Отрезки с $\text{darkRatio}_j < 0.65$ отбрасываются.
5. Скоринг:
$$\text{score}_j = 10\ln(1 + |L_j|) + 6.0\,\bar{D}_j + 80\,\text{darkRatio}_j$$

Выбирается $L^* = \arg\max_j \text{score}_j$.

### Е.3.5. Фильтр правдоподобия

Кандидат $(P_1, P_2)$ принимается, если выполняются условия:

$$\|P_2 - P_1\| \geq 14\ \text{пикс.}$$

$$\min(D(P_1),\, D(P_2)) \geq 2.5\ \text{пикс.}$$

$$\frac{D(P_1) + D(P_2) + D(P_m)}{3} \geq 4.0\ \text{пикс.}, \quad P_m = \frac{P_1+P_2}{2}$$

### Е.3.6. Определение вектора направления ворса

Ворс растёт от центра шкурки к краю. Конец отрезка, ближайший к краю, является наконечником вектора:

$$\text{swap} = D(P_1) < D(P_2)$$

$$\vec{v}_{\text{nap}} = \frac{P_{\text{to}} - P_{\text{from}}}{\|P_{\text{to}} - P_{\text{from}}\|}$$

$$\varphi = \text{atan2}(v_y,\, v_x) \in [0°, 360°)$$

где $P_{\text{to}}$ — конец, ближайший к краю мездры, $P_{\text{from}}$ — дальний конец.

## Е.4. Алгоритм заполнения остатка методом Вороного

### Е.4.1. Диаграмма Вороного в зоне

Дана полигональная зона $Z \subset \mathbb{R}^2$ и множество зёрен $\{s_1, \ldots, s_n\}$, размещённых внутри $Z$.

Ячейка Вороного для зерна $s_i$:

$$V_i = \{p \in Z : \|p - s_i\| \leq \|p - s_j\| \ \forall j \neq i\}$$

### Е.4.2. Релаксация Ллойда

Для улучшения равномерности распределения ячеек применяется итеративная релаксация:

На каждой итерации $k$:

$$s_i^{(k+1)} = \frac{1}{|V_i^{(k)}|}\int_{V_i^{(k)}} p\, dp \quad \text{(центроид ячейки)}$$

После $N_{\text{relax}}$ итераций ячейки $V_i$ принимаются как контуры фрагментов (после клиппинга по $Z$ и отсева $|V_i| < A_{\min}$).

### Е.4.3. Регулярная сетка

Альтернативный алгоритм строит прямоугольную сетку с шагом $h$ и углом поворота $\alpha$:

$$P_{ij} = R(\alpha)\begin{pmatrix}i \cdot h \\ j \cdot h\end{pmatrix} + \text{origin},\quad R(\alpha) = \begin{pmatrix}\cos\alpha & -\sin\alpha \\ \sin\alpha & \cos\alpha\end{pmatrix}$$

Ячейки сетки клиппируются по контуру зоны $Z$; ячейки с площадью $< A_{\min}$ исключаются.

## Е.5. Алгоритм наилучшего совмещения куска (bestFit)

### Е.5.1. Постановка задачи

Дан контур целевой области $F \subset \mathbb{R}^2$ и контур куска инвентаря $S \subset \mathbb{R}^2$. Требуется найти трансформацию $T = (R_\theta, \mathbf{t})$ — поворот на $\theta$ из допустимого множества $\Theta$ и смещение $\mathbf{t}$ — такую, что площадь пересечения максимальна:

> **Примечание.** В режиме А под $F$ понимается текущая незанятая область зоны; в режиме Б — уже сформированный `Fragment`.

$$T^* = \arg\max_{\theta \in \Theta,\, \|\mathbf{t}\| \leq t_{\max}} \text{Area}\!\left(F \cap T(S)\right)$$

### Е.5.2. Метрика покрытия

Коэффициент покрытия фрагмента:

$$\rho = \frac{\text{Area}(F \cap T(S))}{\text{Area}(F)} \in [0, 1]$$

Кусок принимается если $\rho \geq \rho_{\min}$, где $\rho_{\min}$ — параметр конфигурации `InventoryLayoutConfig` (например, 0.85). При нескольких кандидатах выбирается $\arg\max \rho$.

> **Примечание.** Параметр `seamAllowanceReserveMm` (из `NormalizeRules`) задаёт геометрический отступ для припуска на шов и применяется при нормализации контуров фрагментов — он не определяет порог $\rho_{\min}$.

## Е.6. Система координат и перевод пикселей в мм

### Е.6.1. Система координат

Во всех алгоритмах программного комплекса принята стандартная растровая система координат:
- ось X направлена вправо,
- ось Y направлена **вниз** (от верхнего края изображения),
- углы отсчитываются **по часовой стрелке** от оси X.

Угол направления ворса $\varphi = \text{atan2}(v_y, v_x) \in [0°, 360°)$ — в этой же системе.

### Е.6.2. Перевод пикселей в миллиметры

Сканирование производится с разрешением $\text{DPI}$ (типично 200 или 300 точек/дюйм). Коэффициент масштабирования:

$$k = \frac{25.4}{\text{DPI}} \quad \text{мм/пиксель}$$

Пример: при DPI = 200, $k = 0.127$ мм/пикс. Координаты контуров, экспортируемых в поле `scrapContour.path` (`units: "mm"`), умножены на $k$.

## Е.8. Алгоритм NFP+SA (v1) — автоматический подбор кусков

### Е.8.1. Постановка задачи

Дана зона $Z \subset \mathbb{R}^2$ и инвентарь $\mathcal{P} = \{p_1, \ldots, p_n\}$ — набор кусков меха с контурами и направлениями ворса. Требуется найти набор трансформаций $\{(x_i, y_i, \theta_i)\}$, при котором ядра кусков максимально покрывают зону $Z$, перекрытия минимизируются функцией штрафа, а направления ворса не выходят за заданный допуск.

**No-Fit Polygon (растровая аппроксимация).** Классически NFP для пары полигонов $(Z, p_i)$ задаёт множество допустимых позиций $p_i$ относительно $Z$. В растровой реализации NFP аппроксимируется через маски занятости: допустимые позиции куска определяются по отношению к `zoneMask`, а пересечение с `occupiedMask` учитывается в функции энергии как штраф за перекрытие. Это позволяет вычислять допустимые и предпочтительные позиции на каждой итерации SA без построения полигонального NFP в явном виде.

### Е.8.2. Растровая дискретизация

Зона и куски представляются в растровой сетке с шагом $r = 3$ мм:

$$\text{spec} = \left\{ n_x = \left\lceil\frac{W_Z}{r}\right\rceil,\; n_y = \left\lceil\frac{H_Z}{r}\right\rceil \right\}$$

$$\text{zoneMask}[i][j] = \begin{cases}1, & \text{центр ячейки} \in Z \\ 0, & \text{иначе}\end{cases}$$

$$\text{zoneCells} = \sum_{i,j} \text{zoneMask}[i][j]$$

Покрытие решения $R$:

$$\text{cov}(R) = \frac{\left|\bigcup_{p \in R} \text{mask}_p \cap \text{zoneMask}\right|}{\text{zoneCells}}$$

### Е.8.3. Построение начального жадного решения (Greedy Warm Start)

Куски сортируются по убыванию площади. Для каждого куска $p_k$ выполняется 16 попыток случайного размещения:

$$\text{pos}^* = \arg\max_{\text{pos}} \left|\text{mask}_{p_k}(\text{pos}) \cap \overline{C}\right|$$

где $C$ — маска уже покрытых ячеек, $\overline{C} = \text{zoneMask} \setminus C$.

Принятые размещения формируют начальное решение $R_0$.

### Е.8.4. Имитация отжига (Simulated Annealing)

**Функция энергии:**

$$E(R) = -\text{covCells}(R) + \lambda \cdot \text{overlapCells}(R) - \mu \cdot |R|$$

где $\lambda$ — штраф за перекрытие клеток, покрытых двумя и более кусками; $\mu$ — весовой коэффициент, стимулирующий включение куска в решение.

**Расписание температур:**

$$T_k = T_0 \cdot \alpha^k, \quad T_0 = 2.0,\; \alpha = 0.9995,\; T_{\min} = 0.01$$

**Ходы:**

| Ход | Описание |
|-----|---------|
| TRANSLATE | Смещение куска на $(\Delta x, \Delta y) \sim \mathcal{U}(-s, s)^2$ |
| ROTATE | Поворот на $\Delta\theta \sim \mathcal{U}(-\varphi_{\text{tol}}, \varphi_{\text{tol}})$ |
| SWAP | Замена куска на другой из инвентаря |
| ADD | Добавление нового куска |

**Критерий принятия (Metropolis):**

$$P(\text{accept}) = \begin{cases}1, & \Delta E < 0 \\ e^{-\Delta E / T_k}, & \Delta E \geq 0\end{cases}$$

**Остановка:** $T_k < T_{\min}$ или истечение $t_{\max}$.

### Е.8.5. Постобработка фрагментов

После SA получено решение $R^* = \{(p_i, x_i, y_i, \theta_i)\}$. Для каждого куска:

**Ядро куска** (с учётом трансформации размещения):
$$\text{core}_i = T_i\!\left(\text{inset}(p_i,\; a_{\text{allowance}})\right) \cap Z$$

где $T_i(·)$ — трансформация $(x_i, y_i, \theta_i)$, применяемая к исходному контуру куска.

**Фрагмент** (неперекрывающаяся часть):
$$\text{frag}_i = \text{core}_i \setminus \bigcup_{j < i} \text{core}_j$$

Если $\text{frag}_i = \emptyset$ (ядро полностью перекрыто) — кусок исключается из результирующего набора фрагментов, но может сохраняться в диагностике решения как неэффективное размещение.

Если $\text{frag}_i$ состоит из нескольких связных компонент — каждая становится отдельным фрагментом.

**Контроль занятого пространства** (для предотвращения щелей):

$$\text{occupied}_{i+1} = \text{occupied}_i \cup \text{core}_i \quad \text{(исходное ядро, до вычитания)}$$

### Е.8.6. Непокрытые области и коэффициент покрытия

Непокрытые области — участки зоны, не покрытые ни одним ядром:

$$\text{holes} = Z \setminus \bigcup_i \text{core}_i$$

Коэффициент покрытия ядрами:

$$\rho_{\text{core}} = 1 - \frac{\text{Area}(\text{holes})}{\text{Area}(Z)}$$

Подбор считается успешным при достижении заданного порога покрытия: $\rho_{\text{core}} \geq \rho_{\text{target}}$, где $\rho_{\text{target}}$ задаётся параметрами сценария или принимается близким к 1 с учётом технологических зазоров.
