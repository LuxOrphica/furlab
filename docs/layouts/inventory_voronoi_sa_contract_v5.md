# inventory_voronoi_sa — Контракт v5.0

**Файл реализации:** `furlab-web-plugin/src/services/solvers/voronoi_sa_solver.js` (+ `voronoi_sa_output.js`, `voronoi_sa_postprocess.js`, `voronoi_sa_raster.js`, `inventory_voronoi_sa/index.js`)
**Что это:** durable-определение КОРРЕКТНОГО поведения солвера. Рамки, инварианты, приёмка, запреты, диагностика причин неудачи.
**Дата:** 2026-06-27 · **Заменяет:** v4.3 (и все предыдущие v2–v4.x).
**Авторское основание:** 5 принципов, согласованных с владельцем продукта (мин ≥ 70; эстетика обязательно; припуск — инструкция раскрою, не геометрия; подбор кусков по форме, а не по размеру; без вращения).

## 0. Что изменилось против v4.3 (кратко)

| Область | v4.3 | v5.0 |
|---|---|---|

| Выбор N и кусков | N = f(medCoreArea всех selectedPieces); tilePool = N крупнейших | N = f(зона, медианная ячейка Lloyd); куски подбираются под ячейки по **соответствию формы** (aspect ratio + IoU при оптимальном сдвиге), не «самые большие» |

| Покрытие covF | → 100% обязательно (R10) | **→ 100% — цель по умолчанию**; при недостижимости — диагностика причины (баг солвера vs insufficient_input) |
| Приёмка | один статус `ok`/`failed` | **4 статуса:** `ok` / `partial` / `insufficient_input` / `failed` |
| Длина контракта | 9 страниц, внутренние противоречия | 3 страницы, без противоречий |

## 1. Модель (определения)

Солвер раскладывает **ядра** скрап-кусков в зону так, чтобы их фрагменты (пересечение ядра с назначенной территорией) **тайлили зону встык**, без дыр, с эстетикой швов Вороного.

| Термин | Определение |
|---|---|
| **Скрап** | физический кусок меха, `scrapContour`. Уникален, используется ≤ 1 раза. **Не вращается** (фикс. ориентация) |
| **Ядро** `core` = `alignedCoreContour` | `inset(scrapContour, allowanceMm)`. **Единственная геометрия, с которой работает солвер.** Кусок = ядро |
| **Припуск** `allowanceMm` | инструкция раскрою: «дорисовать 12 мм наружу от ядра». **В солвере не участвует** — ни в размещении, ни в покрытии, ни в валидации. Применяется отдельной пост-операцией при экспорте плана кроя |
| **Территория** | дизъюнктная область разбиения зоны, назначенная одному ядру. Строится по Вороному (ближайший seed Lloyd) с коррекцией: клетка назначается куску, чьё ядро её **реально накрывает по полигону** (см. §3, R2) |
| **Фрагмент** = `inZoneCoreContour` | `core ∩ territory ∩ zone`. **Единица покрытия, валидации, приёмки.** Дизъюнктен в метрике (Union без двойного счёта), даже если ядра физически перекрылись на швах |
| **Шов** | место, где два ядра физически перекрываются. В раскрое это место сшивается (2 припуска складываются на изнанку). В метрике — учитывается один раз (Union) |

```
core_i     = inset(scrapContour_i, allowanceMm)     // единственная геометрия в солвере
territory_i = assigned by Voronoi + polygon-coverage (R2)
fragment_i = core_i ∩ territory_i ∩ zone             // дизъюнктен в метрике
// припуск: в солвере отсутствует; при раскрое — grow(core, allowanceMm) наружу
```

**Ключевое упрощение против v4.x:** нет двойственности «ядро vs тело». Кусок = ядро. Припуск — внешний параметр раскроя. Это устраняет класс ошибок v4 (фейк-покрытие по телам, INV4_FAIL и т.д.).

## 2. Входы / выходы

**Вход:** `candidates[]` (`id | inventoryTag`, `scrapContour`, `napDirectionDeg` — вестигиально, не констрейнт), `options` (§6).

**Выход (реплейабельно):** `zone.points`, `effectiveOptions` (вкл. `seed`), `candidates` (с контурами), `placements[]`, `algorithmTrace`, `metrics`, `diagnostics` (причины неудачи).

**Placement:** `inventoryTag`, `alignedCoreContour` (ядро — единственная геометрия покрытия/валидации), `inZoneCoreContour` (фрагмент — юнит покрытия), `rawTerritoryContour` (территория), `physicalMissingMm2`. Поле `alignedContour` = тело с припуском (`grow(core, allowanceMm)`) — только для рендера/отображения в UI; **солвер вычисляет его, но не использует ни в одном вычислении.** Любое использование `alignedContour` или `pts` внутри солвера — баг.

## 3. Инварианты (обязаны держаться всегда)

| # | Требование |
|---|---|
| **R1 — юнит покрытия** | Покрытие, дыры, валидация — по **фрагментам** `inZoneCoreContour`, полигонально. НЕ по полным ядрам, НЕ по растру |
| **R2 — полнота партиции** | `Union(фрагменты) ∩ zone == Union(ядра) ∩ zone` (до под-сетки). Любая точка зоны, накрытая хотя бы одним ядром, **обязана** достаться ровно одному фрагменту. Назначение клетки уважает **реальное накрытие ядром по полигону** (не ближайший seed Lloyd независимо от накрытия) |
| **R3 — дизъюнктность в метрике** | Фрагменты в `Union` считаются без двойного учёта. **Физическое перекрытие ядер на швах разрешено** (это будущий шов), но в покрытии — Union |
| **R4 — припуск — внешний** | `allowanceMm` появляется в солвере **только один раз**: инсет `scrapContour → core` на входе. Больше нигде. В покрытии, валидации, min-size, размещении — отсутствует. Тело (`grow(core, allowance)`) рисуется отдельно при экспорте раскроя |
| **R5 — min-size обязательно** | MBR-short и MBR-long **фрагмента** ≥ `minWidthMm`/`minLengthMm` (по полигону, convex hull → rotating calipers, без припуска). **0 нарушителей в выводе.** Суб-мин фрагмент не создаётся вовсе — перераспределяется при построении (не постфильтром) |
| **R6 — уникальность и фикс. ориентация** | Каждый `inventoryTag` ≤ 1 раза. Куски **не вращаются** |
| **R7 — детерминизм** | Тот же `seed` → побитово тот же вывод. Фиксированный конструктивный порядок и тай-брейки (`tag asc → idx asc`) на всех фазах. Гарантированное завершение, возврат **лучшего по покрытию-фрагментами** состояния |
| **R8 — absorb разрешён с guard'ами** | Absorb (поглощение дыры ядром соседа) — **легальный инструмент**, не гейминг. Два guard'а: (а) фрагмент-поглотитель после absorb имеет MBR ≥ 70, (б) сама поглощаемая дыра имеет MBR ≥ 70 (физически влезет кусок). Без guard'ов — запрещён |
| **R9 — честность метрик** | `coveragePercent` = полигональное покрытие по фрагментам. Самоотчёт солвера — **информативен**. Три источника (солвер = invariants = независимый верификатор) **сходятся**; расхождение = «метрика врёт» |

**Связь R5 ↔ R2 (конструктивно):** если при построении фрагмент получается < 70 — его клетки **перераспределяются** соседям (с guard'ом R8), а не удаляются. Это устраняет partition-gaps (R2) и не создаёт суб-мин (R5) одновременно.

## 4. Алгоритм

### Этап 0 — Pre-flight проверка зоны
1. Эрозия зоны на `-35 мм`. Если после эрозии остались компоненты с MBR-short < 70 → **горло < 70** → `resultStatus = insufficient_input`, репорт с координатами горла. Солвер не запускается.
2. `zoneArea = area(zonePoints)` — полигонально (shoelace).
3. Растеризация зоны для SA-счёта (шаг `gridStepMm = 3`).

### Этап 1 — Подготовка кусков
1. Парсинг `scrapContour`, центрирование.
2. `core = inset(scrapContour, allowanceMm)`. Если инсет схлопывается (< 3 точек) — кусок отбрасывается (не fallback к полному контуру, как в v4.x).
3. Фильтр по min-size (MBR-short/long ядра).
4. `areaMm2 = area(core)`, `aspectRatio = MBR-long / MBR-short`.

### Этап 2 — Выбор N и построение ячеек Lloyd
1. **N подбирается от зоны, не от кусков.** Строится N Lloyd-ячеек зоны (без кусков, только seeds) для нескольких значений N (например, N ∈ {round(zoneArea / c) для c ∈ {5000, 7000, 9000, 12000, 15000}}).
2. Для каждого N — медианная площадь ячейки `medCellArea(N)` и медианный aspect ratio `medCellAR(N)`.
3. Выбирается N, при котором `medCellArea(N) ∈ [0.7, 0.9] × medCoreArea` (ячейка чуть меньше ядра — ядро её закрывает с нахлёстом на соседей).
4. Lloyd-итерации до сходимости.

### Этап 3 — Подбор кусков под ячейки (по форме, не по размеру)
1. Для каждой ячейки вычисляется её aspect ratio.
2. Для каждого свободного куска вычисляется **fitness** к ячейке:
   - **Первичный:** `|AR(cell) - AR(core)|` (близость aspect ratio).
   - **Вторичный (тай-брейк):** `IoU(core, cell)` при оптимальном сдвиге ядра внутри ячейки (без вращения).
3. Жадное назначение: ячейки сортируются по убыванию площади, каждая ячейка получает кусок с лучшим fitness среди свободных. Tie-break: `tag asc → idx asc`.
4. Куски, не назначенные ни одной ячейке, остаются в пуле для SA-ходов ADD.

### Этап 4 — Размещение и SA-оптимизация
1. Начальное размещение: центр ядра = центроид назначенной ячейки.
2. **SA-цикл активен** (не Lloyd-tiling). Ходы: ADD (новый кусок на крупнейший uncovered blob), SWAP, TRANSLATE, REMOVE. **ROTATE отключён** (R6).
3. Энергия: `E = 1000 × (zoneCells - coveredCells) + 8 × overlapCells + 100 × subMinCount + 1 × N` (лексикографически: покрытие > overlap > sub-min > число кусков).
4. `maxIterations = 20000` (детерминированный выход). Cooling обязан использовать бюджет. Возврат **лучшего** состояния.
5. `findLargestUncoveredBlobCentroid` — ход ADD кладёт кусок туда.

### Этап 5 — Построение фрагментов и absorb
1. Назначение клеток: **уважая накрытие ядром по полигону**. Клетка идёт к куску, чьё ядро её реально накрывает (если несколько — ближайший центр; если ни одного → `physMissing`).
2. `fragment = core ∩ territory ∩ zone`.
3. **Absorb (R8):** если дыра (cell без владельца) накрыта ядром соседа И фрагмент соседа после absorb ≥ 70 MBR И сама дыра ≥ 70 MBR — отдать дыру соседу. Без guard'ов — оставить physMissing.
4. Sub-min фрагменты (MBR < 70): клетки перераспределяются соседям (R5), не удаляются.

### Этап 6 — Метрики и диагностика
1. `coveredRatio = area(Union(фрагменты) ∩ zone) / area(zone)`.
2. Дыры классифицируются (см. §5).
3. `resultStatus` — 4 состояния (см. §5).
4. Эстетика-прокси (см. §5).

## 5. Приёмка

### 5.1 Покрытие и дыры

```
coveredRatio = area( Union(inZoneCoreContour) ∩ zone ) / area(zone)     // ≤ 1.0
residual     = Difference( zone, Union(фрагменты) )
realHole     = компонент residual, выживающий после buffer(−1.5 мм)     // под-сеточный мусор прощаем
```

Дыры классифицируются:
- **sliver** — эрозия -1.5 мм схлопывает → прощается (лесенка сетки).
- **partition-gap** — накрыта ядром ≥ 1 соседа, но не отдана фрагменту → баг солвера (нарушение R2).
- **physMissing** — не накрыта ни одним ядром.

physMissing далее классифицируется:
- **closable** — в инвентаре есть свободный подходящий кусок (≥ 70 MBR, по форме подходит) → баг солвера (SA не смог добавить).
- **insufficient_inventory** — в инвентаре нет подходящего свободного куска → свойство входа, не баг.

### 5.2 Четыре статуса результата

| `resultStatus` | Условие | Действие |
|---|---|---|
| `ok` | covF ≥ 99.5% ∧ R2 PASS ∧ R5 PASS ∧ R6 PASS ∧ эстетика PASS | готово к раскрою |
| `partial` | covF ∈ [95%, 99.5%) ∧ R2 PASS ∧ R5 PASS ∧ R6 PASS ∧ physMissing ≤ 1% ∧ эстетика PASS | готово с предупреждением; технолог визирует |
| `insufficient_input` | (а) горло зоны < 70 (pre-flight) ИЛИ (б) physMissing > 1% и причина — insufficient_inventory (нет свободных подходящих кусков) | не баг солвера; чинить зону или инвентарь |
| `failed` | R2 FAIL (partition-gap) ИЛИ R5 FAIL (суб-мин в выводе) ИЛИ R6 FAIL (дубли) ИЛИ covF < 95% при наличии свободных подходящих кусков | баг солвера; чинить алгоритм |

### 5.3 Эстетика (4 прокси, абсолютные пороги)

| Прокси | Формула | Порог PASS | Что ловит |
|---|---|---|---|
| **Выпуклость** | `area(fragment) / area(convexHull(fragment))` | ≥ 0.55 | надрезанные куски |
| **Заполнение MBR** | `area(fragment) / area(MBR(fragment))` | ≥ 0.55 | тонкие языки при широком теле |
| **Доля языков** | `area(fragment \ buffer(fragment, -minWidth/2)) / area(fragment)` | ≤ 0.08 | тонкие отростки |
| **Прямота швов** | `length(sharedBoundary) / straightLength(sharedBoundary)` | ≥ 0.85 | рваные швы |

Каждый фрагмент проверяется по всем 4 прокси. **Эстетика PASS** = ≥ 95% фрагментов проходят все 4 порога. Пороги калибруются машинно: запустить солвер на 5-10 зонах, взять 25-й перцентиль распределения каждого прокси как нижнюю границу «приемлемого».

### 5.4 Инварианты-сверка (R9)

Три источника метрик сходятся (±0.01%):
1. Монитор солвера (внутренний счёт).
2. `invariants` блок в выводе (полигональный пересчёт).
3. Независимый верификатор (shapely/pyclipper, отдельная программа).

Расхождение = «метрика врёт», не «округление».

## 6. Границы параметров

| Параметр | Роль / ограничение |

## 7. Запрещено

- Покрытие/best по полным ядрам (без инсета) или по растру — только по фрагментам, полигонально.
- **Использование `pts` / `alignedContour` / `fullMask` внутри солвера в любом вычислении** — покрытии, маске, absorb, clip, postprocess. `pts` существует только для рендера. Кусок в солвере = `corePts`.
- Списание внутреннего остатка в «растровый артефакт».
- Фантомные метрики (поле, сверяемое само с собой; жёстко нулевой остаток).
- Реюз размещённого скрапа (дубли R6).
- Суб-мин фрагмент в выводе (R5) — перераспределять при построении.
- Absorb **без guard'ов** R8 (без проверки MBR фрагмента-поглотителя и MBR дыры).
- Вращение кусков (R6).
- NFP-перебор при больших N (вешает).
- Время-выход без гарантии завершения.
- Возврат последнего состояния вместо лучшего.
- Проверка покрытия/принадлежности **только по растровой маске** (полигон ядра — истина, R2).
- Подгонка под конкретные зоны (overfitting).
- **Fallback ядра к полному контуру** при схлопывании инсета (баг P0 v4.x) — кусок отбрасывается.

## 8. Известные ограничения

- Под-сеточные периметровые слайверы (~0.05–0.1%) по границам фрагментов — артефакт сетки, прощаются эрозией. **Это единственный прощаемый недокрыт.**
- Инсет сильно вогнутых скрапов срезает >> периметр×allowance — eligibility таких кусков ниже. Кусок может быть отброшен.
- **Горло зоны < 70** — ≥ 70-тайлинг там невозможен; pre-flight детектит и возвращает `insufficient_input`.
- **Insufficient inventory** — если подходящие куски закончились, часть зоны остаётся physMissing; это `insufficient_input`, не баг.
- Эстетика на сложных зонах (с большим числом узких краёв) может не достичь порогов даже при covF = 100% — это `partial`, технолог визирует.

## 9. Воспроизводимость и верификация

**Независимый верификатор** (shapely/pyclipper, **отдельно** от кода солвера) читает run-output и считает с нуля:
- `covF` — покрытие по фрагментам.
- **R2** — полнота партиции (`Union(фрагменты)∩zone == Union(ядра)∩zone`).
- **R3** — дизъюнктность в метрике (Union без двойного счёта; физическое перекрытие на швах разрешено).
- **R5** — MBR-short/long каждого фрагмента (через convex hull → rotating calipers).
- **R6** — счётчик `inventoryTag`, угол каждого placement = 0.
- **R8** — absorb-операции: фрагмент-поглотитель ≥ 70 MBR, дыра ≥ 70 MBR.
- **R9** — сходимость трёх источников.
- Эстетика-прокси (§5.3) на каждом фрагменте.

Печатает сырые числа и места дыр, **падает на любом нарушении жёстких осей** (R2, R5, R6, R8). Функцию покрытия с солвером не разделять.

**Детерминизм** (R7) — хэшем `placements` на повторе сида. Сид-свип ≥ 10 на каждой тестовой зоне.

**Тестовые зоны:** `tests/voronoi-sa/oracle_case_zone_*.json` — фиксированные входы. Verifier и seed-list — человеко-владеемы, агенту read-only.

## 10. Что убрано из v4.3

- **R10 «полное покрытие зоны»** — заменено на «цель 100% + диагностика причины неудачи» (§5.2). 100% не гарантировано для insufficient_input.
- **R11 «эстетика = эталон чистого Вороного»** — заменено на абсолютные пороги 4 прокси (§5.3), калибруемые машинно. Эталон чистого Вороного на реальном инвентаре недостижим.
- **«Любая зона ≥ 70-покрываема»** — отвергнуто как вера, не доказательство. Pre-flight проверяет горла < 70 (§4 Этап 0).
- **Запрет absorb** — заменён на absorb с guard'ами R8.
- **Двойственность «ядро vs тело»** — устранена. Кусок = ядро; припуск — инструкция раскрою.
- **`_lloydTiling: true` всегда** — SA-ветка включена (§4 Этап 4).
- **«N крупнейших кусков»** — заменено на подбор по форме (§4 Этап 3).
- **Краевое исключение «кольцо припуска»** (v4.2) — устарело, припуск не в солвере.
- **Фантомные поля** `rasterSeamArtifactMm2`, `geomCoveragePct=0`, `bestRasterCoveragePct=100` — удалены.
