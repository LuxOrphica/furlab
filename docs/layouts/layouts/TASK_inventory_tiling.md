## 1. Цель — повторить скорняка
Скорняк кладёт зону **регулярно**: немного соразмерных кусков, ровные швы, без дыр, каждый ≥70, ядро до линии. **Чистая раскладка и ЕСТЬ корректная — конфликта между красотой и корректностью нет.** Следствие: уродство (тонкие языки, надрезанные слайверы, рваные швы) = **сигнал дефекта**, не компромисс.

## 2. Принцип
- **Корректность — ЖЁСТКИЕ ограничения:** каждый кусок ≥70 (обе грани MBR) **по построению**; тайлинг полный до границы (край — свисанием); дизъюнктно; без повторов; R2=0; physMissing=0 везде; covF→~100%.
- **Эстетика — ЦЕЛЬ, максимизируемая в рамках:** регулярные гладкие швы, выпуклые/чистые куски, минимум и баланс. **Искажённый Вороной/Lloyd — эстетический каркас** (даёт регулярность); жёсткие ограничения его искажают.
- **Не конфликтуют:** среди корректных раскладок предпочитаем самую регулярную; уродство = дефект, не «зато закрыли».

## 3. Что солвер обязан решить (подзадачи; механизм — за агентом)
Алгоритм должен выдать раскладку по §2. **КАК строит — проектирует агент** (дизайн-пропозал, §5); архитектуру не предписываем, жизнеспособную **выбирает результат на zone_1**. Конкретно решить:
1. **Регулярный тайлинг** зоны на ячейки, **каждая ≥70** (обе грани), доходящие до границы **везде** (край — свисанием).
2. **Каждая ячейка обеспечена ровно одним скрапом** из инвентаря, чьё ядро её кроет; без повторов.
3. **Полное покрытие:** каждая клетка зоны — в одном фрагменте (R2=0, physMissing=0), включая край.
4. **Регулярность/чистота максимальны** (без языков, надрезов, рваных швов); кусков минимум, соразмерны.
5. **Без зазоров ПО ПОСТРОЕНИЮ (не починкой):** размещение/раздача обязаны сразу не оставлять голых стыков — подбор скрапа под ячейку есть **часть построения**, а не проход-ремонт по дырам. Если выбранный скрап не даёт ≥70/полного покрытия ячейки — **построение выбирает другой скрап на этом же шаге**, а не размещает-и-латает потом. **Отдельной фазы «закрыть дыры» нет.** Появилась дыра в результате → дефект построения (см. §4), чинится в раздаче, не патчем.
6. **Корректность контура:** внешний контур по **signed** area (не `|area|`).
7. **Детерминизм:** фикс. порядки и тай-брейки; тот же seed → тот же выход.

## 4. Приёмка (PASS = всё ниже; иначе НЕ PASS)
**Судит независимый верификатор (shapely), а НЕ самоотчёт солвера** (`resultStatus`/метрики солвера — информативны, не приёмка).
1. **Каждый фрагмент ≥70 ПО ПОСТРОЕНИЮ:** absorb = 0 (любой absorb или дроп-с-потерей-покрытия = дефект).
2. **R2 = 0; physMissing = 0 ВЕЗДЕ** (covF→~100%; прощается только под-сеточный сливер, эрозия −1.5; край закрыт свисанием).
3. **Без повторов; дизъюнкт ≈ 0.**
4. **Без уродства (эстетика):** четыре измеримых прокси — выпуклость `area/convexHull`, заполнение `area/MBR`, **язык** (доля площади куска вне эрозии на ½·minWidth — ловит тонкие отростки при широком теле), прямота швов (длина границы между кусками / прямое расстояние). **Эталон порогов — ЧИСТЫЙ Вороной** зоны (геометрия зоны + N семян + Lloyd, БЕЗ скрапов и БЕЗ ≥70): это верхняя планка регулярности. Порядок: (1) посчитать прокси на чистом Вороном zone_1/2/4 → эталонные распределения; (2) те же прокси на раскладке солвера; (3) **порог = эталон минус допустимый отход**, отход задаёт владелец (сколько ухудшения ради ≥70/покрытия терпим). НЕ брать порог из искажённой (нашей) раскладки — это сравнение с самим собой. **Сперва меряем и репортим** → **жёсткий FAIL после подтверждения порогов** (Qwerty). Принцип: уродство = дефект.
5. **Детерминизм** (тот же seed → тот же выход). Пока в механизме остаётся seed-зависимость (SA-размещение) — **сид-свип ≥ 10**: PASS держится на ≥10 разных seed, а не на одном удачном (защита от везения, ср. zone_1 с фрагментом 70.03). Если раздача станет полностью детерминированной (от seed не зависит) — сид-свип отпадает.
6. **Независимый пересчёт (R9):** числа солвера = числа верификатора.

*Замечание:* если у зоны есть **горло <70** (часть уже минимума) — ≥70-тайлинг там невозможен; verify это **детектит и падает с диагнозом** (по домену такого нет, но не зацикливаться).
