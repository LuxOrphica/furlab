## 1. Концепция

Solver раскладывает **ядра** скрап-кусков так, чтобы они **тайлили зону встык** (как ячейки Вороного), без зазоров. Припуск в раскладке не участвует — он дорисовывается наружу постпроцессом для плана раскроя.

**Определения (незыблемо):**
- **Скрап** — физический кусок меха, `scrapContour`. Уникален, используется ≤ 1 раза.
- **Ядро** (`core`) = `scrapContour`, инсетнутый на `allowanceMm`. Это **видимая** часть, которая реально кроет зону (припуск на шве уходит на изнанку и не виден). **Ядро — единица раскладки, покрытия и валидации.**
- **Территория** (`cell`) — ячейка Вороного, назначенная одному ядру. Территории **дизъюнктны** и в сумме покрывают зону.
- **Фрагмент** = `core ∩ cell ∩ zone` — то, что физически режется. Фрагменты тайлят (не пересекаются).
- **Тело** (`body`) = `core + allowanceMm` наружу = исходный `scrapContour`. **Появляется ТОЛЬКО в финальном выводе** как кусок для раскроя. Тела соседей перекрываются на стыках на ~2×allowance — это и есть **шов** (там сшивают, припуск складывается на изнанку). Тело **не участвует** в размещении, покрытии и валидации.

```
core_i   = inset(scrapContour_i, allowanceMm)      // единица раскладки
cell_i   = территория Вороного куска i             // дизъюнктна
fragment = core_i ∩ cell_i ∩ zone                  // что режется, тайлит
body_i   = grow(core_i, allowanceMm)  [постпроцесс] // кусок для раскроя, перекрытие = шов
```

**Цель:** ядра кроют зону без внутренних дыр и без фрагментов уже `minWidth`, каждый скрап — один раз. SA достигает этого сам, размещая ядра (включая края зоны); патч-фаз нет.

**Ошибки v3.1, которые v4 исправляет (от первых принципов):**
- v3.1 тайлил ТЕЛА (фрагмент = тело∩ячейка) и мерил покрытие по ним → ядра оказывались утоплены на припуск, а метрика показывала «99.x» при дырявых ядрах. v4: тайлятся ядра, покрытие по ядрам.
- v3.1 закрывал дыры absorption'ом, переиспользуя уже размещённые скрапы и плодя сливеры → дубли и суб-минимумы, маскированные припуском. v4: патч-фаз нет, дыры честные.

### `candidates[]` (обязательно)

  id | inventoryTag,            // уникальный идентификатор скрапа
  scrapContour,                // [{x,y}...] или строка — ПОЛНЫЙ контур скрапа (тело)

### `options`

| Поле | Тип | По умолчанию | Описание |
|---|---|---|---|
| `napTarget` | number | 0 | Целевое направление ворса зоны, ° |
| `napTol` \| `napTolDeg` | number | 15 | Допуск ворса, ° |
| `seed` | number | 1 | Seed воспроизводимости |
| `maxIterations` | number | 20000 | **Детерминированный стоп SA.** Бюджет итераций |
| `allowanceMm` | number | UI: 12 | Припуск. **Только**: инсет `scrapContour → core` на входе и grow `core → body` в постпроцессе. Больше нигде |
| `minWidthMm` | number | UI: 70 | Мин. узкая сторона ядра (MBR-short). Фильтр кандидатов И валидация фрагментов |
| `minLengthMm` | number | UI: 70 | Мин. длинная сторона ядра (MBR-long) |
| `maxPieces` | number | `[задать]` | Жёсткий потолок числа кусков (защита от дробления/зависания) |
| `plateauEps` / `plateauWindow` | number | `[задать]` | Стоп добора кусков при плато покрытия (Δ < eps за window шагов) |
| `zoneHoles` | Array | [] | Дыры в зоне |
| `onProgress` | function | null | Колбэк прогресса |

> **Удалено из v3.1:** всё, что касается патч-фаз и припуск-в-маске: `absorptionCriterion`, `overhangMm` (если был только для absorption/targeted), `mosaicMode`, `seamAllowanceReserveMm`. Time-based выход (`maxIterations=null`) — убрать; выход всегда итерационный.

### min-size — единый смысл (исправление двойственности v3.1)
`minWidthMm`/`minLengthMm` применяются **к ЯДРУ**, по короткой/длинной грани **минимального повёрнутого прямоугольника** (rotating calipers), НЕ по bbox и НЕ по площади:
1. **Фильтр кандидатов:** ядро с MBR-short < `minWidthMm` или MBR-long < `minLengthMm` отсеивается до SA.
2. **Валидация фрагментов:** фрагмент (core∩cell∩zone) ниже порога по MBR — нарушение (см. §6, ось 3). Не растворять переназначением (это патч-фаза); SA не должен такое порождать.

> bbox запрещён: повёрнутый фрагмент 22×140 даёт bbox ~114×114 и ложно проходит. Площадной порог тоже слаб (лента 250×20 проходит). Только MBR-short.

## 3. Алгоритм

### Этап 0 — Зона
1. `zoneArea = area(zonePoints)` — **полигональная** (shoelace), не растровая.
2. Растеризация для внутреннего SA-счёта (шаг `gridStepMm = 3`) допустима, но **итоговое покрытие меряется полигонально по ядрам** (§5), не растром.

### Этап 1 — Куски
1. Парсинг `scrapContour`, центрирование.
2. `core = inset(scrapContour, allowanceMm)` (ClipperOffset, масштаб `×SC`).
   > Инсет на сильно вогнутых контурах срезает >> периметр×allowance — свойство офсета, учитывать при eligibility. Это **единственное** место инсета.
3. Фильтр по min-size (MBR-short/long ядра).
4. `areaMm2 = area(core)` — реальная площадь ядра.

### Этап 2 — Preflight
`Cmed` = медиана `areaMm2` ядер; `Nbase = ceil(zoneArea / Cmed)`; в `selectionDebug`.

### Этап 3 — Размещение (SA, детерминированно)
- Состояние: набор размещённых ядер + seed'ы Вороного → дизъюнктные территории, тайлящие зону.
- Энергия включает покрытие **по ядрам** (`Union(core∩cell)∩zone`), overlap (=0 целевой), и штрафы за: невалидные фрагменты (MBR < min), повтор скрапа.
- **Уникальность:** скрап, уже использованный в состоянии, не может быть добавлен второй раз (ни одной фазой, ни одним ходом).
- **Края:** ходы SA обязаны уметь добавлять ядра/seed'ы к границе зоны (там, где краевые дыры). С **потолком `maxPieces`** и **стопом по плато** покрытия.
- **Выход (детерминированный):** по `maxIterations` или сходимости/плато; cooling обязан использовать бюджет (не вылетать на `Tmin` слишком рано). **Возвращать ЛУЧШЕЕ по покрытию состояние**, не последнее. `exitReason` — реальная причина. Никакого растрового `fullCoverage100`.
- **Никаких патч-фаз** (gap_fill, absorption, targeted). Дыру, которую нельзя закрыть свободным валидным скрапом, оставить дырой.

### Этап 4 — Постпроцесс (припуск, только для отчёта)
Для каждого размещённого ядра: `body = grow(core, allowanceMm)` наружу. Тела соседей перекрываются на стыках — это шов. Тело идёт в вывод как кусок для раскроя; на метрики/валидацию не влияет.

## 5. Метрика покрытия

```
coveredRatio = area( Union(все alignedCoreContour) ∩ zonePoints ) / area(zonePoints)
```
- Числитель — **объединение ЯДЕР** (перекрытие учитывается один раз), пересечённое с зоной. НЕ тела, НЕ `Σ` площадей.
- Знаменатель — полигональная площадь зоны. `coveredRatio ≤ 1.0` всегда; >100% = баг подсчёта.

**Дыры:**
```
residual    = Difference(zonePoints, Union(ядер))
realHoles   = компоненты residual, выживающие после buffer(−1.5 мм)  // под-сеточные слайверы прощаем
            классифицировать: interior / perimeter
```

---

## 6. Инварианты = 4 оси успеха

| Ось / инвариант | Проверка | PASS |
|---|---|---|
| 1. Покрытие | `Union(ядер)∩zone / zone` | как можно выше (реалистично, без дробления) |
| 2. Реальные дыры | компоненты `zone − Union(ядер)` после `buffer(−1.5)` | **interior = 0**, perimeter минимален |
| 3. Валидность фрагментов | MBR-short каждого ядра ≥ `minWidthMm` (и long ≥ `minLengthMm`) | **0 нарушителей** |
| 4. Уникальность скрапа | счётчик `inventoryTag` | **0 дублей** |
| Дизъюнктность | попарное пересечение фрагментов | ≈ 0 |
| `napValid` | угол куска в ±`napTol` | честный |

> **Урок v3.1 (сохранить):** инвариант, сравнивающий поле сам с собой (ядро=фрагмент → «ячейка ∈ ядро» всегда true), даёт фиктивный PASS. Каждый инвариант обязан сравнивать ДВА РАЗНЫХ источника в согласованной геометрии. Внутренним числам solver'а не доверять — мерить независимо (§8).

## 7. Известные ограничения

1. **Под-сеточные слайверы (~0.05–0.1%).** Границы фрагментов по сетке 3 мм дают лесенку; остаток < ~1.5 мм по эрозии — артефакт, не дыра (шов перекрывает). Приёмка по реальным дырам (после `buffer(−1.5)`), не по сырому residual.
2. **Краевое покрытие может быть неполным.** Если край физически нечем закрыть свободным валидным скрапом — он остаётся дырой (честно). Лучше честные 97% валидными ядрами, чем 100% сотней слайверов.
3. **Инсет вогнутых ядер.** ClipperOffset(−allowanceMm) на вогнутых срезает >> периметр×allowance — eligibility таких кусков занижена; на богатом инвентаре не мешает.
4. **Voronoi nearest-center невзвешенный** — ячейки не пропорциональны размеру ядра.
5. **napTol не проверяется при SWAP/ADD** (наследие; вынести в энергию).

## 8. Воспроизводимость и тестирование

### Независимый верификатор (отдельный скрипт, НЕ код solver'а)
shapely/pyclipper, читает run-output, считает **с нуля** все 4 оси + дизъюнктность, печатает сырые числа, падает громко на любой оси. Не шарить функцию покрытия с solver'ом.

- **Детерминизм:** тот же seed дважды → побитово идентичный вывод (хэш placements).
- **Сид-свип:** ≥ 10 сидов — распределение всех 4 осей (универсальность vs везение).
- **Многозонье:** zone_1 / zone_2 / zone_4.
- **Три источника** (монитор solver'а = harness = независимый pyclipper) обязаны сходиться. Расхождение = метрика врёт.

### Критерий приёмки
```
на фикс-seed:
  assert realHolesInterior == 0
  assert subMinFragments == 0           // MBR-short по ядру
  assert duplicateScraps == 0
  assert overlap(fragments) ≈ 0
  assert coveredRatio <= 1.0
  assert три источника метрик сходятся
  coveredRatio — как можно выше, но НЕ ценой нарушения осей 2/3/4
```

## 9. Что НЕ реализовано / зарезервировано

- Взвешенный Voronoi (Power Diagram) — ячейки пропорциональны площади ядра. (Двухфазная SA→Lloyd спроектирована в v3.0, признана вредной/ненужной — Lloyd оптимизирует центроидальную энергию, не покрытие, и открывает дыры. **Не возвращать без жёсткого гейта по реальному покрытию.**)
- Площадной/area-критерий eligibility вместо точечного.
- napTol в SWAP/ADD; penalty за NAP в энергии.
- 180°-разворот для короткого ворса.

## 10. ТЕКУЩЕЕ СОСТОЯНИЕ И ПОРЯДОК ВОССТАНОВЛЕНИЯ

> Этот контракт — целевая модель. Реализация ей сейчас НЕ соответствует.

**Состояние: СЛОМАНО.** Попытка добрать края через NFP + перебор числа кусков повесила прогон (NFP попарно O(n²) при ~100 кусках) и развалила выкладку. **Чистого/рабочего коммита для отката НЕТ.**

**Порядок (детали — в отдельном ТЗ, §0 «редукция к якорю»):**
1. Собрать рабочий якорь **редукцией**: выпилить NFP/перебор-кусков и патч-фазы; оставить детерминированный SA, тайлящий ядра и **завершающийся**. Якорь определяется поведением (завершается + верификатор: чистые фрагменты), а не коммитом. Закоммитить.
2. Дальше — только маленькие обратимые шаги от якоря; каждый шаг: прогон → завершился? → верификатор (4 оси не хуже)? → коммит / откат. Никаких больших переписываний.
3. Инкрементально добирать краевое покрытие в SA с `maxPieces` и стопом по плато.
