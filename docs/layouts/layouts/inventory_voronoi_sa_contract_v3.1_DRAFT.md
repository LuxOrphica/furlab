## 1. Концепция

Солвер размещает скрап-куски в зону методом имитации отжига (SA), назначает каждой растровой
ячейке зоны покрывающий её кусок (restricted Voronoi), строит фрагменты как пересечение куска с
его территорией, затем полигональной absorption переназначает непокрытые ячейки физически
покрывающим соседям.

```
fragment_i = piece_i ∩ territory_i
```

где `territory_i` — объединение растровых ячеек, назначенных куску `i` по правилу
«ближайший центроид среди физически покрывающих».

**Что это даёт:** границы фрагментов следуют ячейкам Вороного (эстетические швы), при этом
непокрытые места («швы на стыке швов», где ячейка проигрывает по расстоянию всем соседям
одновременно) закрываются absorption — переназначением физически покрывающему соседу, а не
добавлением материала.

**Ключевое отличие от v3.0:** дыры закрываются *переназначением* (материал уже на месте),
а не gap-fill-латанием. На инвентаре с избытком площади (наблюдалось 5.5×) реальной нехватки
материала (VOID) не возникает — все дыры absorbable.

### 2.1 Двойной смысл `minWidthMm` / `minLengthMm`

## 3. Алгоритм

### Этап 0 — Подготовка зоны
1. Rasterize `zonePoints` с шагом **`gridStepMm` = 3 мм** → `zoneMask`
2. Вычесть `zoneHoles`
3. `zoneArea = area(zonePoints)` — **полигональная площадь** (shoelace), НЕ растровая
   > Изменение против v3.0, где использовалось `zoneCells × 9`. Растровое приближение
   > (`zoneAreaRaster`) сохраняется в selectionDebug для справки, но знаменатель coverage —
   > полигональный. Это устранило зависимость знаменателя от seed.

### Этап 1 — Подготовка кусков
1. Парсинг `scrapContour` → `rawPts`
2. Центрирование относительно centroid
3. `corePts = inset(centeredPts, allowanceMm)` через ClipperOffset (если allowanceMm > 0)
   > **Критично:** `corePts` сохраняется в placement и используется как `alignedCoreContour`.
   > В v3.0 это поле ошибочно равнялось фрагменту. Инсет масштабируется на scale-фактор Clipper
   > (`-allowanceMm × SC`), не `-allowanceMm`. `[CONFIRM масштаб]`
   > **Известное поведение:** на вытянутых/вогнутых кусках ClipperOffset(−12) срезает существенно
   > больше, чем периметр×12 (наблюдалось до 66% площади у сильно вогнутых) — это свойство офсета
   > на невыпуклых контурах, не баг. Следствие: eligibility вогнутых кусков по core занижена;
   > на богатом инвентаре не мешает. `[CONFIRM / решить, нужен ли иной метод запаса]`
4. Фильтр по `minWidthMm`/`minLengthMm` (см. §2.1)
5. `areaMm2 = area(полигона)` — **реальная площадь**, НЕ bbox
   > Изменение против v3.0 (bbox). Влияет на Cmed, сортировку greedy, preflight.

### Этап 2 — Preflight (статистика для Monitor)
- `Cmed` — медиана реальных `areaMm2`
- `Nbase = ceil(zoneArea / Cmed)`
- selectionDebug → output

### Этап 3 — Растеризация масок (трёхбитовая)

> **Изменение против v3.0.** Маска многобитовая, потому что у неё несколько потребителей с
> разными требованиями к строгости:

Для каждого куска два растра — `mask` (по core-контуру) и `fullMask` (по полному контуру).
Каждый бит:

| Бит | Значение | Критерий | Потребитель |
|-----|----------|----------|-------------|
| bit0 (`& 1`) | центр ячейки внутри контура | строгий | SA-энергия (coverage, overlap) |
| bit1 (`& 2`) | ≥1 из 5 точек (4 угла + центр) внутри | либеральный | (boundary eligibility) |
| bit2 (`& 4`) | ≥3 из 5 точек внутри (~60%) | большинство | **absorption eligibility** |

> **Эволюция критерия absorption (история сессии):** bit0 (центр) терял ячейки на стыках мелких
> кусков; bit1 (любой угол) раздувал территории (кусок «претендовал» на ячейку, задев углом, →
> большие незакрытые территории, всплеск числа кусков 25→35); bit2 (≥3/5, majority) — компромисс.
> **Известное ограничение:** bit2 — это «большинство точек», не «большинство площади»; шаг
> дискретности 20%. На узких стыках в 1-2 ячейки может промахиваться. Честный критерий —
> `area(cell ∩ piece)/area(cell) ≥ 0.5`. На текущем инвентаре crit=bit0 и crit=bit2 дают
> идентичный результат (разница нулевая), поэтому переход на площадной критерий не приоритетен.
> `[CONFIRM семантика битов в коде]`

### Этап 4 — IFP (Inner Fit Polygon), два кеша

> **Изменение против v3.0.** Два IFP вместо одного:

```
ifpTight = MinkowskiDiff(zonePoints, corePts)                       // overhang=0
ifpWide  = MinkowskiDiff(inflate(zonePoints, overhangMm), corePts)  // overhang>0
```

- **tight** — для SA и greedy warm start. Кусок не выходит за зону; энергетика плотная.
- **wide** — для targeted-фазы. Кусок может свисать за кромку на `overhangMm` (свес — норма для
  меха, обрезается по кромке).

> **Обоснование (A/B сессии):** общий overhang для SA деградирует покрытие (SA тратит итерации на
> бесполезные позиции в зоне свеса; на фикс-seeds Δ был стабильно отрицательным). IFP-сплит
> нейтрализует деградацию (Δ=0 побитно на фикс-seeds), оставляя свес доступным targeted-фазе.

### Этап 5 — Greedy Warm Start
Сортировка по реальной `areaMm2` DESC, жадное размещение, 16 попыток на кусок, позиции из tight-IFP.
Угол `normalizeDeg(napTarget − napDeg) ± dAngle` в пределах ±napTol.

### Этап 6 — SA loop

**Энергия (минимизируем):**
```
E = 1000 × (zoneCells − coveredCells) + 8 × overlapCells + 1 × N
```
coverage/overlap считаются по **bit0** масок (строгий критерий — SA-поведение стабильно).
Лексикографический приоритет: покрытие > перекрытия > число кусков.

**Температура:** `T0 = max(E×0.05, zoneCells×0.5)`, `Tmin = T0×0.0005`, `alpha = 0.9975`.

**Ходы:** TRANSLATE 38%, ROTATE 20%, SWAP 14%, REMOVE 12%, ADD остаток.

> **Известные ограничения (наследие v3.0, не закрыты):**
> - napTol проверяется только при ROTATE; SWAP/ADD ставят угол жёстко (ограничение №5 v3.0).
>   Рекомендация: сэмплировать угол из ±napTol во всех ходах. `[не реализовано]`
> - Поворот используется только в коридоре ±napTol (это и есть допуск по ворсу — материальное
>   ограничение, не лень). Зеркалирование запрещено (у шкурки есть лицо). 180° допустимо только
>   для короткого ворса — флаг материала, не реализован.

**Стоп:** `T < Tmin`, или `maxIterations` (если задан, для A/B), или таймаут. `exitReason` в trace.

### Этап 6.5 — Мультирестарт

> **Новый этап.** Межсидовый разброс покрытия (наблюдалось 98.6–99.97% на одном входе) бьётся
> рестартами, а не доводкой одной раскладки.

```
для i in 1..numRestarts:
    результат_i = SA(seed_i)   // бюджет maxSolveMs/numRestarts
выбрать результат с минимальным residualInterior (ДО absorption)
```

> **Наблюдение:** на текущем инвентаре все seeds дают absorbable-раскладку (VOID=0), поэтому
> мультирестарт не является фильтром нехватки — но поднимает медиану покрытия и снижает остаток
> до absorption. `[CONFIRM numRestarts по умолчанию]`

### Этап 7 — Voronoi Assignment (restricted)

> **Изменение против v3.0** (там было глобальное nearest-center + gap-fill).

```
для каждой ячейки зоны:
    covering = { j : кусок j физически покрывает ячейку }   // по fullMask
    assignment[cell] = argmin_{j ∈ covering} dist(cell_center, centroid_j)
```

Назначение **только среди физически покрывающих** кусков. Territory_j = union назначенных ячеек.

### Этап 8 — Построение фрагментов (физическое)

> **КРИТИЧНОЕ изменение против v3.0.** Фрагмент и Union строятся из физических пересечений,
> НЕ из территорий.

```
fragment_j = Clipper.Intersection(alignedContour_j, territory_j) ∩ zonePoints
```

- `alignedContour_j` — **полный** контур куска (с припуском), не core.
- Результат клипуется точным `zonePoints` (гладким полигоном), не растровой маской.

> **Почему полный контур, а не core:** покрытие зоны мерит, накрыта ли зона мехом физически.
> Припуск — это тоже мех (уходит под шов). Core (`alignedCoreContour`) используется только для
> eligibility и инварианта шва, НЕ для границы фрагмента и НЕ для метрики покрытия.

> **Почему это устранило фантомные 100%:** в v3.0 в Union клались территории; территория
> недотянутого куска покрывала зону, давая `residual=0` (ложь), хотя кусок физически туда не
> доставал → дыра видна на канвасе. Флаг `isTerritoryPlaceholder` (для случая `intersectedArea < 1`)
> ловил только полное недотягивание, частичное оставалось. Физический Union убирает класс целиком —
> непокрытая часть территории автоматически попадает в residual.

### Этап 9 — Absorption (multi-pass)

> **Новый этап, заместивший gap-fill. Ключевой для достижения покрытия.**

```
covered = Union(физических фрагментов)        // БЕЗ placeholder-территорий
while (pass < maxPass):                        // maxPass ~5 [CONFIRM]
    absorbed = 0
    для каждой непокрытой ячейки:
        eligible = { j : fullMask_j[idx] & 4 }   // физически покрывает (majority)
        если eligible непусто:
            назначить ближайшему по центроиду j; absorbed++
    пересобрать фрагменты задетых кусков (Union ячеек)
    если absorbed == 0: break
```

> **Критично — порядок:** absorption работает по **физическому** покрытию (`fullMask`), запускается
> **после** физического Union (этап 8), видит реальные непокрытые ячейки. В неверном порядке
> (absorption до физического Union) она видела бы территории как покрытие и пропускала дыры.

> **Что absorption закрывает / не закрывает (по классификации дыр):**
> - дыра под соседями (under > 50%) → переназначается, закрывается;
> - дыра под пустотой (VOID, under < 5%) → не закрывается, остаётся честным residual.
> На текущем инвентаре VOID=0 → absorption закрывает всё.

### Этап 10 — Dissolve мелких фрагментов
```
while (pass < 10):
    для фрагментов ниже порога (§2.1), по площади ASC:
        ячейки → ближайшему eligible соседу (fullMask & 4)
        если все розданы: удалить placement
        иначе: status = "under_threshold" (не считается за изменение)
```
Растворённый фрагмент исключается из переобработки (`phase="dissolved"`).

### Этап 11 — Targeted-цикл (реализован, спящий)

> **Новый этап. На текущем инвентаре patchCount=0 — это НОРМА, не сбой.**

После absorption, для residual-компонентов interior:
- транзакция: snapshot → ADD/TRANSLATE кандидата у центроида компонента (из wide-IFP) →
  полный хвост конвейера → если residualInterior не упал, откат.

> **Когда активируется:** только когда после assignment + absorption остаётся компонент,
> физически не покрытый ни одним куском (локальная нехватка материала, VOID > 0). Такой случай
> в серии тестов не наблюдался (инвентарь с избытком). `patchCount: 0` при VOID=0 — корректное
> поведение: absorption закрыла всё раньше.
> **Узкие трещины:** компонент уже `minWidth` нельзя закрыть отдельным фрагментом (dissolve съест) —
> для них TRANSLATE соседа, не ADD. Логика ширины компонента выбирает ветку. `[TRANSLATE-ветка
> реализована? CONFIRM]`

## 5. Метрика покрытия

> **КРИТИЧНОЕ изменение против v3.0.**

```
coveredRatio = area(Clipper.Union(все inZoneContour) ∩ zonePoints) / area(zonePoints)
```

- Числитель — **объединение** фрагментов (перекрытие учитывается ОДИН раз), пересечённое с зоной.
- НЕ `Σ inZoneAreaMm2` — сумма допускает двойной счёт перекрытий и контуров за границей зоны,
  давала фантомные >100% (наблюдалось 100.67%).
- Знаменатель — полигональная площадь зоны.

> **Гарантия:** `coveredRatio ≤ 1.0` всегда. Если метрика показывает >100% — баг подсчёта
> (сумма вместо объединения), не реальное перекрытие.

**residual:**
```
residual = Clipper.Difference(zonePoints, Union(fragments))
компоненты ≥ 18 мм² классифицируются:
  - у кромки зоны (< 50% за эродированной на 1.5×gridStep границей) → residualPerimeter
  - иначе → residualInterior
```

## 7. Известные ограничения

1. **Растровые швы (~0.07%).** Границы фрагментов — union ячеек 3 мм, дают лесенку; между
   фрагментами остаётся остаток ~275 мм² (32 компонента ≤17 мм²) — `rasterSeamArtifactMm2`.
   Это НЕ дыры покрытия (шов физически перекрывает). **Порог приёмки: 99.8%, не 100%.**
   Регрессионный тест: `residualInterior < 300 мм²` (≈0.1% zoneArea), НЕ `== 0`.
   Устранение (Douglas-Peucker simplify ~3-4 мм, или полигональные Вороные-границы) — косметика
   швов, отложено, риск новых артефактов.

2. **Растровое приближение площадей.** `gridStepMm = 3` — погрешность ~1-3% для мелких/вогнутых зон.
   `zoneArea` — полигональный (точный); растровым остаётся только внутренний SA-счёт покрытия.

3. **Voronoi nearest-center невзвешенный** (не Power Diagram). Ячейки не пропорциональны размерам
   кусков. Крупный кусок получает геометрически такую же ячейку, как мелкий.

4. **Вогнутые куски: инсет ядра.** ClipperOffset(−allowanceMm) на сильно вогнутых/вытянутых
   контурах срезает >>периметр×allowance (до 66%). Eligibility таких кусков по core занижена.
   На богатом инвентаре не мешает (есть замена). `[решить, нужен ли иной метод запаса]`

5. **absorptionCriterion bit2 — дискретность.** «≥3 из 5 точек» ≈ 60% по точкам, не по площади,
   шаг 20%. На узких стыках возможен промах. На текущем инвентаре эффект нулевой (crit=bit0=bit2).

6. **napTol не проверяется при SWAP/ADD** (наследие v3.0).

7. **Targeted спит при VOID=0.** На инвентаре с избытком материала absorption закрывает всё,
   targeted patchCount=0 — норма. Активируется при локальной нехватке (не наблюдалось).

## 8. Воспроизводимость и тестирование

> **Новый раздел.**

### Oracle-кейс = INPUT (replayable)
Экспорт «run output» сохраняет: `zonePoints` + все `candidates` с контурами + `effectiveOptions`
(включая seed) + placements с тремя контурами + `uncoveredComponents` с `pts`. Этого достаточно
для побитового реплея и для регрессионного фикстура.

> **Расхождение, выявленное в сессии:** старая кнопка «Экспорт case» сохраняла output с легаси-
> параметрами (`rPreview:10, maxIter:300, maxPieces:48`) — невоспроизводимо. Заменена на
> «Экспорт run output» с `effectiveOptions`. **Старую кнопку убрать или переключить на
> effectiveOptions.** `[CONFIRM]`

### Headless A/B-харнесс
`scripts/ab_harness.js` гоняет солвер через тот же HTTP API (`POST /api/layout/modes/preview` →
`server.js` → `voronoiSaSolver.solve()`), что и UI — НЕ параллельная реализация. Считает
physical coverage + under/void классификацию через ClipperLib. Поддерживает фикс-seed и
переключение `absorptionCriterion` для A/B.

> **Верифицировано:** на одном seed монитор, харнесс и независимый pyclipper-расчёт совпали
> бит-в-бит (cov, residual, число компонентов). Телеметрии можно доверять.

### Критерий приёмки (регрессионный тест)
```
на фикс-seed (input-кейс):
  assert VOID == 0
  assert residualInterior < 300 мм²        // НЕ == 0 (растровые швы)
  assert coveredRatio <= 1.0               // защита от двойного счёта
  assert noOverlaps: overlap_inZone == 0
  assert три источника метрик сходятся
```

## 9. Что НЕ реализовано (зарезервировано)

- Взвешенный Voronoi (Power Diagram) — ячейки пропорциональные areaMm2. **Двухфазная схема
  (SA → Lloyd-refinement + notContained energy) была спроектирована, но не понадобилась:**
  restricted assignment + absorption достигают цели без выпуклых power-ячеек. Сложность Power
  Diagram (выпуклость ячеек vs вогнутые куски → E=0 недостижим на реальном скрапе) делает его
  нецелесообразным, пока absorption справляется.
- Площадной критерий eligibility (`area(cell∩piece)/area(cell)`) вместо дискретных битов.
- TRANSLATE-ветка targeted для узких трещин `[CONFIRM статус]`.
- napTol в SWAP/ADD; penalty за NAP-отклонение в энергии.
- Эрозия-порог для dissolve (вместо площадного minAreaMm2).
- 180°-разворот для короткого ворса (флаг материала).
