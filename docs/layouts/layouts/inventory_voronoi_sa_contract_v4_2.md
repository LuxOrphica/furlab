## 1. Модель (определения)

Солвер раскладывает **ядра** скрап-кусков так, чтобы их обрезанные по территориям **фрагменты тайлили зону встык**, без зазоров. Припуск в раскладке не участвует.

| Термин | Определение |
|---|---|
| **Скрап** | физический кусок, `scrapContour`. Уникален, используется ≤ 1 раза |
| **Ядро** `core` = `alignedCoreContour` | `inset(scrapContour, allowanceMm)` — видимая кроющая часть. Полные ядра могут наезжать друг на друга (сырьё) |
| **Территория** | дизъюнктная область разбиения зоны, назначенная одному ядру (в экспорте может не выводиться) |
| **Фрагмент** = `inZoneCoreContour` | `core ∩ территория ∩ zone` — **то, что режется и тайлит. ЕДИНИЦА ПОКРЫТИЯ, ВАЛИДАЦИИ, ПРИЁМКИ** |
| **Тело** `body` = `alignedContour` | `grow(core, allowanceMm)` наружу = исходный `scrapContour`. **Только** в финальном выводе для раскроя; перекрытие соседних тел = шов. На покрытие интерьера не влияет (см. Ось2Б — край) |

```
core_i     = inset(scrapContour_i, allowanceMm)          // полное ядро (могут наезжать)
fragment_i = core_i ∩ territory_i ∩ zone                 // дизъюнктно, тайлит, ЭТО покрытие
body_i     = grow(core_i, allowanceMm)  [постпроцесс]    // кусок для раскроя
```

Раздел территорий **конструктивен**: minWidth (R5) обеспечивается ПРИ построении, а не постфильтром. Конкретный порядок раздачи (напр. от краёв внутрь) — выбор реализации, см. TASK; контракт его не предписывает.

## 2. Входы / выходы (рамки данных)

**Вход:** `candidates[]` ( `id|inventoryTag`, `scrapContour`, `napDirectionDeg` ), `options` (см. §6).

**Выход (реплейабельно):** `zone.points` (обязательно), `effectiveOptions` (включая `seed`), `candidates` (с контурами), `placements[]`, `algorithmTrace.phaseA`, `metrics`.

**Placement:** `inventoryTag`, `phase`, `alignedCoreContour` (ядро), `inZoneCoreContour` (фрагмент — юнит покрытия), `rawTerritoryContour` (территория, если выводится), `alignedContour` (тело — только раскрой), `physicalMissingMm2`.

## 3. Ключевые требования (инварианты — обязаны держаться всегда)

| # | Требование |
|---|---|
| **R1 — юнит покрытия** | Покрытие, дыры, валидность, best-state — по **фрагментам** `inZoneCoreContour`, полигонально. НЕ по полным ядрам, НЕ по телам, НЕ по растру |
| **R2 — полнота партиции** | `Union(фрагменты) ∩ zone == Union(ядра) ∩ zone` (с точностью до под-сетки). Любая точка зоны, накрытая хотя бы одним размещённым ядром, обязана достаться ровно одному фрагменту. Накрытых-но-ничьих спотов быть не должно |
| **R3 — дизъюнктность** | Фрагменты попарно не пересекаются (overlap ≈ 0); тайлят встык. Гарантируется тем, что назначение клетка→владелец — **функция** (ровно один владелец) |
| **R4 — припуск только на границах** | `allowanceMm` появляется ровно дважды: инсет `scrapContour→core` на входе и grow `core→body` в постпроцессе. В размещении, покрытии, валидности, min-size — отсутствует |
| **R5 — min-size (инвариант ПОСТРОЕНИЯ)** | MBR-short **ФРАГМЕНТА** ≥ `minWidthMm` (и MBR-long ≥ `minLengthMm`), по полигону фрагмента (**convex hull → rotating calipers**), НЕ bbox/площадь/ядро, без припуска. **Суб-мин фрагмент не создаётся вовсе:** если регион даёт владельцу фрагмент <minWidth, он ПРИ ПОСТРОЕНИИ либо утолщается/отдаётся валидному соседу (по **полигональному** покрытию ядра, не по растровой маске), либо не выделяется в кусок. Это инвариант построения, **не постфильтр** |
| **R6 — уникальность** | Каждый `inventoryTag` использован ≤ 1 раза |
| **R7 — детерминизм и выход** | Тот же `seed` → побитово тот же вывод. Детерминизм распространяется на **конструктивный порядок**: обход кусков/клеток и все тай-брейки фиксированы (напр. `tag asc → idx asc`), BFS-инициализация в растровом порядке — на каждой фазе построения и локального ремонта. Выход гарантированно завершается, возвращает **лучшее по покрытию-фрагментами**. Растрового выхода нет |
| **R8 — без гейминга (build-time vs пост-хок)** | **Запрещён пост-хок гейминг**: реюз размещённого скрапа ради дыры, накрутка/фантомные метрики, NFP-перебор-взрыв, silent drop без NOTE, отдельные `absorption/gap_fill/targeted` ФАЗЫ-патчи поверх готовой раскладки ради инфляции покрытия. **Разрешён build-time конструктив/ремонт**: формирование и локальная починка территорий В ПРОЦЕССЕ построения, дающие **реальное валидное покрытие** (по полигону ядра, без реюза скрапов, без накрутки). Тест по сути, не по имени: *реюзает скрап? плодит сливер? поднимает метрику без реального покрытия? обходит верификатор?* → запрещено |
| **R9 — честность метрик** | Reported `coveragePercent` = полигональное покрытие по фрагментам. Внутренний остаток НИКОГДА не списывается в «растровый шовный артефакт». Артефакт — только под-сеточный **периметровый** слайвер (гибнет при `buffer(−1.5)`). Фантомных метрик нет. Три источника (монитор солвера = `invariants` = независимый верификатор) сходятся |

**Связь R5 ↔ R2 (конструктивно):** полнота держится тем, что построение назначает **каждую** накрытую ядром клетку владельцу, **чьё ядро её реально кроет (по полигону)**. Когда тонкий регион утолщается/поглощается, его клетки **переназначаются** валидному накрывающему соседу (BFS), а не осиротевают. Только клетки, которые ни один валидный сосед не накрывает по ядру, остаются `physMissing` (не суб-мин кусок).

## 4. Метрика и классификация дыр

```
coveredRatio = area( Union(inZoneCoreContour) ∩ zone ) / area(zone)        // ≤ 1.0 всегда
realHole     = компонент дыры, выживающий после buffer(−1.5 мм)            // под-сеточный мусор прощаем
```

Дыры классифицируются по **двум разным осям**, с разной мерой и разным порогом:

- **Ось2А — внутренние, по ЯДРАМ.** `zone − Union(фрагменты)`, dist ≥ INTERIOR_DIST. **Закрываемые** (накрыты ядром, можно тайлить) = **0**, гарантируется R2. **Истинный interior-physMissing** (накрыт только несмежно/overlap-reach, центры клеток вне всех ядер) — **репортится с местами (NOTE)**; жёсткий порог пока НЕ фиксируется (ставится из мульти-зонных данных). Телом интерьер **не кредитуется** (наезд тел = двойной слой = маскировка).
- **Ось2Б — краевые, по ТЕЛАМ.** `zone − Union(alignedContour)`, dist < INTERIOR_DIST. Истинный physMissing, не достающийся **даже припуском** соседнего тела. **PASS ≤ допуска** (провизорно ≤ 1% зоны; домен-настраиваемо; **места печатаются всегда**, финальную приемлемость визирует технолог). Обоснование: край панели — это шов с соседней деталью, припуск тела там — реальный материал, а не «спрятанный под изнанку».

Природа дыры:
- **partition-gap** — место накрыто ядром, но не отдано фрагменту (нарушение R2). Закрывается **конструктивным назначением** клетки накрывающему владельцу, без нового материала.
- **physMissing** — место не накрыто ни одним ядром (`physicalMissingMm2`>0). Истинная нехватка.

---

## 5. Приёмка (по фрагментам; интерьер — ядра, край — тела)

1. **Покрытие** `Union(фрагменты)∩zone / zone` — максимально, без дробления.
2. **Ось2А (интерьер, ядра):** закрываемые внутренние дыры **0**; interior-physMissing — NOTE + места.
3. **Ось2Б (край, тела):** краевой physMissing **≤ допуска**, места печатаются.
4. **Валидность (R5):** MBR-short фрагмента ≥ `minWidthMm` — **0** нарушителей.
5. **Уникальность:** **0** дублей.
+ дизъюнктность ≈ 0 (R3); `coveredRatio ≤ 1.0`; полнота партиции (R2); три источника метрик сходятся (R9); детерминизм спот-реплеем (R7).

Прогон пройден только при выполнении на фикс-сиде → подтверждении сид-свипом (≥10) → на нескольких зонах. **Алгоритм общий; зоны — тест-кейсы на обобщение, под них не подгонять** (зоно-специфичный хак = overfitting, запрещён).

## 6. Границы параметров

| Параметр | Роль / ограничение |
|---|---|
| `allowanceMm` (UI 12) | только R4 |
| `minWidthMm` / `minLengthMm` (UI 70) | только R5 |
| `seed` | воспроизводимость (R7) |
| `maxIterations` | детерминированный потолок SA; cooling обязан использовать бюджет |
| `maxPieces` | жёсткий потолок числа кусков (защита от дробления и зависания) |
| `plateauEps` / `plateauWindow` | стоп добора по плато покрытия |
| `INTERIOR_DIST` | граница интерьер/край (dist до контура зоны); согласован с `gridStepMm` |
| порог реальной дыры | эрозия `buffer(−1.5 мм)`, согласована с `gridStepMm` |
| допуск Ось2Б | провизорно ≤1% зоны; ставится/уточняется из мульти-зонных данных |
| cap по времени | страховка от зависания; при срабатывании — гарантированный выход с возвратом best |

Конкретные числа калибруются под инвентарь/зоны; границы и смысл — фиксированы здесь.

## 7. Запрещено

Покрытие/best по полным ядрам или растру · списание внутреннего остатка в артефакт · фантомные метрики · реюз размещённого скрапа для закрытия дыр · **создание суб-мин фрагмента** (R5 — инвариант построения) · **пост-хок патч-фазы поверх готовой раскладки ради инфляции покрытия** (build-time конструктив/ремонт — разрешён, см. R8) · NFP-перебор при больших N (вешает) · время-выход без гарантии завершения · возврат последнего состояния вместо лучшего · **проверка покрытия/принадлежности только по растровой маске** (полигон ядра — истина) · **подгонка под конкретные зоны** (overfitting) · оптимизация прокси (центроиды/MBR), открывающая дыры, без гейта по реальному полигональному покрытию.

## 8. Известные ограничения

- Под-сеточные слайверы (~0.05–0.1%) по границам фрагментов — артефакт сетки, прощаются эрозией.
- **covF = 100% недостижимо по инсету:** ядро = `inset(scrap, allowance)`, поэтому краевое кольцо зоны шириной с припуск ядром не кроется никогда — оно кроется **телом** (учитывается Ось2Б). Это не дефект.
- **Interior micro-pockets:** мелкие внутренние карманы на тройных стыках, накрытые только несмежно/overlap-reach (центры вне ядер) — честный мелкий остаток (Ось2А NOTE), связным валидным куском не закрываемы.
- Инсет сильно вогнутых ядер срезает >> периметр×allowance — eligibility таких кусков занижена.
- Часть зоны может остаться непокрытой при истинной нехватке материала (`physMissing`) — честный результат, не дефект.

## 9. Воспроизводимость и верификация

Независимый верификатор (shapely/pyclipper, **отдельно** от кода солвера) читает run-output и считает с нуля: покрытие covF, **R2** (полнота), **R3** (дизъюнктность), **R5** (MBR-short фрагмента через convex hull), **Ось2А** (интерьер по ядрам), **Ось2Б** (край по телам), уникальность, **R9** (сходимость трёх источников). Печатает сырые числа и места дыр, падает на любом нарушении жёстких осей. Функцию покрытия с солвером не разделять. Детерминизм (R7) — хэшем `placements` на повторе сида. Расхождение трёх источников = «метрика врёт», не «округление». Верификатор, фикстуры и seed-list — человеко-владеемы, агенту read-only.
