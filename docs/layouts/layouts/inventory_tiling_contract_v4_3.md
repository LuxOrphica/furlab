# inventory tiling solver — Контракт v4.3

**Файл реализации:** `furlab-web-plugin/src/services/solvers/voronoi_sa_solver.js` (+ `voronoi_sa_output.js`). Переименование ожидается (напр. `inventory_voronoi_tiling`), см. TASK.
**Что это:** durable-определение КОРРЕКТНОГО ВЫХОДА солвера — рамки, инварианты, приёмка, запреты. Специфит ВЫХОД, **не механизм** (механизм/раздача — в `TASK_inventory_tiling.md`).
**Дата:** 2026-06-22 · **Заменяет:** v4.2.

**Изменения v4.2 → v4.3** (по слову эксперта: ядро доходит до линии зоны ВЕЗДЕ; любая зона из лекал ≥70-покрываема; чистая раскладка = корректная):
- **Снято краевое исключение.** Удалён «covF=100% недостижим по инсету» и Ось2Б как «краевой допуск». Край кроется **ядром** (свисающим куском, `overhangMm`), не списывается на тело.
- **Осевая классификация дыр слита.** Нет деления интерьер/край: **любая** дыра, пережившая под-сетку, = дефект (physMissing=0 ВЕЗДЕ). covF → **~100%**.
- **Добавлен R10** (полное покрытие зоны ядрами, нет UNCOV) и **R11** (эстетика — регулярность/чистота, измеримо).
- **absorb=0** как явное требование приёмки; **счёт кусков = ВЫХОД** (не цель).
- Куски **не вращаются** (фикс. ориентация); `nap*`/`angleDeg` — вестигиальные.
- Приёмку судит **независимый верификатор**, не самоотчёт солвера.

## 1. Модель (определения)

Солвер раскладывает **ядра** скрап-кусков так, чтобы их обрезанные по территориям **фрагменты тайлили зону встык, без зазоров, до самой границы**. Припуск в раскладке не участвует. Куски **не вращаются**.

| Термин | Определение |
|---|---|
| **Скрап** | физический кусок, `scrapContour`. Уникален, ≤ 1 раза. Может **свисать за зону** (`overhangMm`), чтобы ядро доходило до границы |
| **Ядро** `core` = `alignedCoreContour` | `inset(scrapContour, allowanceMm)` — видимая кроющая часть. Полные ядра могут наезжать (сырьё) |
| **Территория** | дизъюнктная область разбиения зоны, назначенная одному ядру |
| **Фрагмент** = `inZoneCoreContour` | `core ∩ территория ∩ zone` — **то, что режется и тайлит. ЕДИНИЦА ПОКРЫТИЯ, ВАЛИДАЦИИ, ПРИЁМКИ** |
| **Тело** `body` = `alignedContour` | `grow(core, allowanceMm)` = исходный `scrapContour`. **Только** для раскроя; перекрытие соседних тел = шов. На приёмку покрытия НЕ влияет |

```
core_i     = inset(scrapContour_i, allowanceMm)     // ядро (могут наезжать); свисание → ядро доходит до границы зоны
fragment_i = core_i ∩ territory_i ∩ zone            // дизъюнктно, тайлит зону ЦЕЛИКОМ, ЭТО покрытие
body_i     = grow(core_i, allowanceMm)  [раскрой]   // кусок для кроя
```

Раздел территорий **конструктивен**: minWidth (R5) и полнота (R2,R10) обеспечиваются ПРИ построении. Механизм раздачи (искажённый Вороной/Lloyd-каркас и т.п.) — в TASK; контракт его не предписывает, но требует его **выходных свойств** (ниже).

## 2. Входы / выходы

**Вход:** `candidates[]` (`id|inventoryTag`, `scrapContour`, `napDirectionDeg` — **вестигиально, не констрейнт**), `options` (§6).
**Выход:** `zone.points`, `effectiveOptions` (вкл. `seed`), `candidates` (с контурами), `placements[]`, `algorithmTrace`, `metrics` (**информативны, не приёмка**).
**Placement:** `inventoryTag`, `alignedCoreContour` (ядро), `inZoneCoreContour` (фрагмент — юнит), `rawTerritoryContour` (если выводится), `alignedContour` (тело — раскрой), `physicalMissingMm2`.

## 3. Инварианты (обязаны держаться всегда)

| # | Требование |
|---|---|
| **R1 — юнит покрытия** | Покрытие, дыры, валидность — по **фрагментам** `inZoneCoreContour`, полигонально. НЕ по ядрам/телам/растру |
| **R2 — полнота партиции** | `Union(фрагменты) ∩ zone == Union(ядра) ∩ zone` (до под-сетки). Любая накрытая ядром точка зоны → ровно одному фрагменту. Накрытых-но-ничьих нет |
| **R3 — дизъюнктность** | Фрагменты не пересекаются (overlap ≈ 0); тайлят встык. Назначение клетка→владелец — функция |
| **R4 — припуск только на границах** | `allowanceMm` ровно дважды: инсет `scrap→core`, grow `core→body`. В размещении/покрытии/валидности отсутствует |
| **R5 — min-size (инвариант ПОСТРОЕНИЯ)** | MBR-short и MBR-long **фрагмента** ≥ `minWidthMm`/`minLengthMm`, по полигону (**convex hull → rotating calipers**), без припуска. **Суб-мин фрагмент не создаётся вовсе.** **absorb = 0** (никаких сбросов кусков ради валидности; дроп-с-потерей-покрытия = дефект). Инвариант построения, не постфильтр |
| **R6 — уникальность** | Каждый `inventoryTag` ≤ 1 раза. Куски **не вращаются** (фикс. ориентация) |
| **R7 — детерминизм** | Тот же `seed` → побитово тот же вывод. Фикс. конструктивный порядок и тай-брейки на всех фазах. Гарантированное завершение, возврат лучшего |
| **R8 — без гейминга** | Запрещён пост-хок гейминг (реюз скрапа ради дыры, накрутка/фантомные метрики, silent drop, патч-фазы ради инфляции). Разрешён build-time конструктив/ремонт, дающий **реальное валидное покрытие**. Тест по сути: реюз? сливер? метрика без покрытия? обход верификатора? → запрещено |
| **R9 — честность; гейт = независимый верификатор** | `coveragePercent` = полигональное покрытие по фрагментам. Самоотчёт солвера (`resultStatus`/метрики) — **информативен, не приёмка**; судит **независимый** верификатор. Три источника (солвер = invariants = верификатор) **сходятся**; расхождение = «метрика врёт» |
| **R10 — полное покрытие зоны** | `Union(ядра) ∩ zone == zone` (до под-сетки): **каждая** клетка зоны накрыта хотя бы одним ядром, **включая край** (свисающими кусками). Нет UNCOV внутри зоны. Вместе с R2 ⇒ фрагменты тайлят зону целиком ⇒ **covF → ~100%**, **physMissing = 0 ВЕЗДЕ** |
| **R11 — эстетика** | Швы регулярные/гладкие, куски выпуклые/чистые (без тонких языков, глубоких надрезов, рваных швов), кусков минимум и соразмерны. Измеримо (§5.4). **Уродство = дефект, не компромисс** (чистая раскладка = корректная — они не конфликтуют) |

**Связь R5 ↔ R2/R10 (конструктивно):** полнота держится тем, что построение накрывает каждую клетку зоны ядром (R10, край — свисанием) и назначает её владельцу, **чьё ядро её реально кроет (по полигону)**. Тонкий регион не создаётся (R5). Голых клеток (physMissing) и накрытых-но-ничьих (R2-gap) в зоне не остаётся.

**Счёт кусков = ВЫХОД, не цель:** избыточные куски (полностью накрытые другими) дропать; **добавлять** чистый ≥70-кусок, если дыра его требует. Конкретное число зависит от инвентаря.

## 4. Метрика и классификация дыр

```
coveredRatio = area( Union(inZoneCoreContour) ∩ zone ) / area(zone)     // → ~1.0
realHole     = компонент дыры, выживающий после buffer(−1.5 мм)         // под-сеточный мусор прощаем
```

**Деления интерьер/край НЕТ.** Любой `realHole` (внутри ИЛИ у края) = **дефект построения**. Две природы — обе означают, что РАЗМЕЩЕНИЕ неверно, и устраняются в построении, НЕ отдельным проходом-ремонтом:
- **partition-gap** — место накрыто ядром, но не отдано фрагменту (нарушение R2): раздача обязана назначить клетку накрывающему владельцу при построении.
- **physMissing** — место не накрыто ни одним ядром (нарушение R10): размещение обязано **не оставлять** этого зазора (ядро доходит везде, край — свисанием). Дыра в результате = дефект размещения, а не повод её «залатать».

**Латание запрещено** (см. §7): нет отдельной фазы, которая ходит по дырам и их затыкает. Полнота — свойство построения. Единственное прощаемое — под-сеточный периметровый сливер (гибнет при `buffer(−1.5)`).

## 5. Приёмка (PASS = всё ниже; судит независимый верификатор)

1. **Покрытие covF → ~100%** (`Union(фрагменты)∩zone / zone`); **physMissing = 0 ВЕЗДЕ** (сверх под-сетки); R2-gap = 0.
2. **Валидность (R5):** MBR-short/long фрагмента ≥ min — **0** нарушителей; **absorb = 0**.
3. **Уникальность:** **0** дублей; дизъюнктность ≈ 0 (R3).
4. **Эстетика (R11):** четыре измеримых прокси — выпуклость `area/area(convexHull)`, заполнение `area/area(MBR)`, **язык** (доля площади вне эрозии на ½·minWidth — ловит тонкие отростки), прямота швов (длина шва / прямое расстояние). **Эталон порогов — ЧИСТЫЙ Вороной зоны** (геометрия + N семян + Lloyd, без скрапов и без ≥70) как верхняя планка регулярности; **порог = эталон минус допустимый отход**, отход задаёт владелец контракта. Порог НЕ брать из искажённой (нашей) раскладки — сравнение с собой запрещено. **Сперва меряются и репортятся** → **жёсткий FAIL после подтверждения порогов**. Грубые языки/надрезы/рваные швы = FAIL.
5. **Детерминизм (R7):** спот-реплеем; **сид-свип ≥ 10**.
6. **Сходимость (R9):** числа солвера = числа независимого верификатора.

**Алгоритм общий; зоны — тест-кейсы на обобщение, под них не подгонять** (overfitting запрещён). *Если у зоны есть горло <minWidth (часть уже минимума) — ≥70-тайлинг там невозможен; верификатор это детектит и падает с диагнозом (по домену такого нет).*

## 6. Границы параметров

| Параметр | Роль / ограничение |
|---|---|
| `allowanceMm` (UI 12) | только R4 |
| `minWidthMm` / `minLengthMm` (UI 70) | только R5 |
| `overhangMm` | насколько скрап может свисать за зону, чтобы ядро дошло до границы (R10) |
| `seed` | воспроизводимость (R7) |
| `maxIterations` | детерминированный потолок; cooling обязан использовать бюджет |
| `maxPieces` | **страховочный** потолок (анти-дробление/зависание), **НЕ цель**; число кусков = выход |
| порог реальной дыры | эрозия `buffer(−1.5 мм)`, согласована с `gridStepMm` |
| пороги эстетики (R11) | выпуклость/заполнение/прямота/баланс; ставятся владельцем контракта |
| cap по времени | страховка от зависания; гарантированный выход с возвратом лучшего |

`napTarget`/`napTolDeg`/`angleDeg` — **вестигиальные**, не трактовать как живые ограничения.

## 7. Запрещено

Покрытие/best по ядрам/телам/растру · списание остатка в артефакт · фантомные метрики · реюз размещённого скрапа ради дыр · **создание суб-мин фрагмента / absorb** (R5) · **краевое исключение** («кольцо припуска» — ядро обязано дойти до линии, R10) · **оправдание дыры «честной нехваткой»** (любая зона ≥70-покрываема) · **уродство как компромисс** (R11) · **ЛЮБАЯ отдельная фаза-латание поверх готового размещения** (проход по дырам с absorb/edge-shift/targeted-ADD/U-фазами ради закрытия — полнота есть свойство ПОСТРОЕНИЯ, не пост-обработки) · NFP-перебор при больших N · время-выход без завершения · возврат last вместо best · проверка покрытия/принадлежности **только по растровой маске** (полигон ядра — истина) · **вращение кусков** · подгонка под конкретные зоны (overfitting) · приёмка по самоотчёту солвера (судит независимый верификатор, R9).

## 8. Известные ограничения

- Под-сеточные периметровые слайверы (~0.05–0.1%) по границам фрагментов — артефакт сетки, прощаются эрозией. **Это единственный прощаемый недокрыт.**
- Инсет сильно вогнутых скрапов срезает >> периметр×allowance — eligibility таких кандидатов ниже (влияет на подбор, **не оправдывает дыру**).

*(Удалено в v4.3: «covF=100% недостижим по инсету» и «часть зоны может остаться непокрытой при истинной нехватке — честный результат» — оба неверны: ядро доходит до линии свисанием, любая зона ≥70-покрываема.)*

## 9. Воспроизводимость и верификация

Независимый верификатор (shapely/pyclipper, **отдельно** от кода солвера) читает run-output и считает с нуля: covF, **R2** (полнота), **R3** (дизъюнктность), **R5** (MBR через convex hull; absorb=0), **R10** (полное покрытие, physMissing=0 везде), **R11** (эстетика-прокси), уникальность, **R9** (сходимость). Печатает сырые числа и места дыр, **падает на любом нарушении**. Функцию покрытия с солвером не разделять. Детерминизм — хэшем `placements` на повторе сида + сид-свип ≥10. Верификатор, фикстуры, seed-list — человеко-владеемы, агенту read-only.
