## 1. Модель (определения)

Солвер раскладывает **ядра** скрап-кусков так, чтобы их обрезанные по территориям **фрагменты тайлили зону встык**, без зазоров. Припуск в раскладке не участвует.

| Термин | Определение |
|---|---|
| **Скрап** | физический кусок, `scrapContour`. Уникален, используется ≤ 1 раза |
| **Ядро** `core` = `alignedCoreContour` | `inset(scrapContour, allowanceMm)` — видимая кроющая часть. Полные ядра могут наезжать друг на друга (сырьё) |
| **Территория** | дизъюнктная область разбиения зоны, назначенная одному ядру (в экспорте может не выводиться) |
| **Фрагмент** = `inZoneCoreContour` | `core ∩ территория ∩ zone` — **то, что режется и тайлит. ЕДИНИЦА ПОКРЫТИЯ, ВАЛИДАЦИИ, ПРИЁМКИ** |
| **Тело** `body` = `alignedContour` | `grow(core, allowanceMm)` наружу = исходный `scrapContour`. **Только** в финальном выводе для раскроя; перекрытие соседних тел = шов. На метрики не влияет |

```
core_i     = inset(scrapContour_i, allowanceMm)          // полное ядро (могут наезжать)
fragment_i = core_i ∩ territory_i ∩ zone                 // дизъюнктно, тайлит, ЭТО покрытие
body_i     = grow(core_i, allowanceMm)  [постпроцесс]    // кусок для раскроя
```

## 2. Входы / выходы (рамки данных)

**Вход:** `candidates[]` ( `id|inventoryTag`, `scrapContour`, `napDirectionDeg` ), `options` (см. §6).

**Выход (реплейабельно):** `zone.points` (обязательно), `effectiveOptions` (включая `seed`), `candidates` (с контурами), `placements[]`, `algorithmTrace.phaseA`, `metrics`.

**Placement:** `inventoryTag`, `phase`, `alignedCoreContour` (ядро), `inZoneCoreContour` (фрагмент — юнит покрытия), `rawTerritoryContour` (территория, если выводится), `alignedContour` (тело — только раскрой), `physicalMissingMm2`.

## 3. Ключевые требования (инварианты — обязаны держаться всегда)

| # | Требование |
|---|---|
| **R1 — юнит покрытия** | Покрытие, дыры, валидность, best-state — считаются по **фрагментам** `inZoneCoreContour`, полигонально. НЕ по полным ядрам, НЕ по телам, НЕ по растру |
| **R2 — полнота партиции** | `Union(фрагменты) ∩ zone == Union(ядра) ∩ zone` (с точностью до под-сетки). Любая точка зоны, накрытая хотя бы одним размещённым ядром, обязана достаться ровно одному фрагменту. Накрытых-но-ничьих спотов быть не должно |
| **R3 — дизъюнктность** | Фрагменты попарно не пересекаются (overlap ≈ 0); тайлят встык |
| **R4 — припуск только на границах** | `allowanceMm` появляется ровно дважды: инсет `scrapContour→core` на входе и grow `core→body` в постпроцессе. В размещении, покрытии, валидности, min-size — отсутствует |
| **R5 — min-size** | По **короткой грани минимального повёрнутого прямоугольника (MBR-short) ФРАГМЕНТА** ≥ `minWidthMm` (и MBR-long ≥ `minLengthMm`). НЕ bbox, НЕ площадь, без припуска |
| **R6 — уникальность** | Каждый `inventoryTag` использован ≤ 1 раза |
| **R7 — детерминизм и выход** | Тот же `seed` → побитово тот же вывод. Выход SA итерационный (`maxIterations`/плато), гарантированно завершается, возвращает **лучшее по покрытию-фрагментами** состояние. Растрового выхода нет |
| **R8 — без патчей** | Нет absorption / gap_fill / targeted-фаз. Дыры не закрываются реюзом уже размещённого скрапа и не плодятся суб-мин слайверы |
| **R9 — честность метрик** | Reported `coveragePercent` = полигональное покрытие по фрагментам. Внутренний остаток НИКОГДА не списывается в «растровый шовный артефакт». Артефакт — только под-сеточный **периметровый** слайвер (гибнет при `buffer(−1.5)`). Фантомных метрик (поле, сверяемое само с собой, или жёстко нулевой остаток) нет. Три источника (монитор солвера = `invariants` = независимый верификатор) сходятся |

**Связь R5 ↔ R2 (критично):** отбраковка фрагмента по min-size (R5) **обязана перераспределять его territory**, а не просто удалять кусок. Клетки выброшенного фрагмента возвращаются в пул и отдаются ближайшему **выжившему** соседу, чьё ядро их накрывает (его фрагмент только растёт → остаётся валидным, дизъюнктность цела). Клетки без накрывающего валидного соседа остаются истинной дырой (`physMissing`). Просто удалить суб-мин кусок, осиротив его territory, = прямое нарушение R2 (накрыто ядрами, но ничьё).

## 4. Метрика и классификация дыр

```
coveredRatio = area( Union(inZoneCoreContour) ∩ zone ) / area(zone)        // ≤ 1.0 всегда
residual     = Difference( zone, Union(inZoneCoreContour) )
realHole     = компонент residual, выживающий после buffer(−1.5 мм)         // под-сеточный мусор прощаем
```
Классификация реальной дыры (две оси):
- **внутр / край** — по расстоянию до контура зоны (касается = край);
- **слайвер / провал** — по доле периметра дыры на контуре зоны и толщине после эрозии. **Тонкий периметровый слайвер** (≳55% периметра на крае, гибнет/худеет при эрозии) — допустим. **Толстый провал** (выпирает в поле) — недопустим, даже если касается края.

Природа дыры:
- **partition-gap** — место накрыто ядром, но не отдано фрагменту (нарушение R2). Закрывается перераспределением территорий, без нового материала.
- **physMissing** — место не накрыто ни одним ядром (`physicalMissingMm2`>0). Истинная нехватка; закрывается добавлением ядра.

## 5. Приёмка (4 оси — ВСЕ по фрагментам)

1. **Покрытие** `Union(фрагменты)∩zone / zone` — максимально, без дробления.
2. **Реальные дыры**: внутренних **0**; толстых провалов **0**; допустимы только тонкие периметровые слайверы.
3. **Валидность** (MBR-short фрагмента ≥ `minWidthMm`): **0** нарушителей.
4. **Уникальность**: **0** дублей.
+ дизъюнктность ≈ 0; `coveredRatio ≤ 1.0`; три источника метрик сходятся (R9).

Прогон считается пройденным только при выполнении на фикс-сиде, затем подтверждённым сид-свипом (≥10) и на нескольких зонах (алгоритм общий, зоны — тест-кейсы).

## 6. Границы параметров

| Параметр | Роль / ограничение |
|---|---|
| `allowanceMm` (UI 12) | только R4 |
| `minWidthMm` / `minLengthMm` (UI 70) | только R5 |
| `seed` | воспроизводимость (R7) |
| `maxIterations` | детерминированный потолок SA; cooling обязан использовать бюджет, не вылетать на `Tmin` раньше |
| `maxPieces` | жёсткий потолок числа кусков (защита от дробления и зависания) |
| `plateauEps` / `plateauWindow` | стоп добора по плато покрытия |
| порог реальной дыры | эрозия `buffer(−1.5 мм)`, согласована с `gridStepMm` |
| cap по времени | страховка от зависания; при срабатывании — гарантированный выход с возвратом best |

Конкретные числа калибруются под инвентарь/зоны; границы и смысл — фиксированы здесь.

## 7. Запрещено

Покрытие/best по полным ядрам или растру · списание внутреннего остатка в артефакт · фантомные метрики · реюз размещённого скрапа для закрытия дыр · суб-мин слайверы · absorption/gap_fill/targeted · NFP-перебор при больших N (вешает) · время-выход без гарантии завершения · возврат последнего состояния вместо лучшего · Lloyd-релаксация без жёсткого гейта по реальному покрытию (оптимизирует центроиды, не покрытие — открывает дыры).

---

## 8. Известные ограничения

Под-сеточные слайверы (~0.05–0.1%) по границам фрагментов — артефакт сетки, прощаются эрозией. · Инсет сильно вогнутых ядер срезает >> периметр×allowance (свойство офсета) — eligibility таких кусков занижена. · Невзвешенный Voronoi не делает ячейки пропорциональными размеру ядра. · Часть зоны может остаться непокрытой при истинной нехватке материала (`physMissing`) — это честный результат, не дефект.

---

## 9. Воспроизводимость и верификация

Независимый верификатор (shapely/pyclipper, **отдельно** от кода солвера) читает run-output и считает с нуля все 4 оси (§5) + R2 (полнота партиции) + R3 (дизъюнктность), печатает сырые числа, падает на любом нарушении. Функцию покрытия с солвером не разделять. Детерминизм (R7) проверяется хэшем `placements` на повторе сида. Расхождение трёх источников метрик трактуется как «метрика врёт», а не «округление».
