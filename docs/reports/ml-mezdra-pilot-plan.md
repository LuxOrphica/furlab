# ML Pilot: Mezdra Contour (Result-first)

Дата: 2026-03-09

## Что уже сделано

Добавлен рабочий каркас ML-пилота в `ui-lab/line-direction-visualizer-react/ml/v3`:
- `prepare_dataset.py` — копирование входных сканов в датасет
- `annotate_grabcut.py` — полуавтоматическая разметка масок мездры
- `train.py` — обучение бинарной U-Net (`mezdra / not-mezdra`)
- `infer.py` — инференс одной картинки, экспорт маски и внешнего контура
- `eval_folder.py` — пакетный прогон по папке, сводный CSV
- `requirements.txt`
- `README.md` с командами

Подготовлены 2 реальных скана в датасет:
- `ml/datasets/mezdra_v1/images/Gemini_Generated_Image_t6n2yot6n2yot6n2.png`
- `ml/datasets/mezdra_v1/images/Gemini_Generated_Image_z1y9llz1y9llz1y9.png`

## Главное правило приемки

Результат принимается только если внешний контур идет по мездре, а не по бахроме.
Стабильность/время вторичны, если контур неверный.

## Что делаем следующим шагом

1. Разметить эти 2 маски вручную через `annotate_grabcut.py`.
2. Добавить еще минимум 6-10 реальных сканов в `images`.
3. Разметить маски для всего набора.
4. Запустить `train.py` и `eval_folder.py`.
5. Сравнить overlay и метрики с текущим V3 stable.

## Минимальный порог для решения GO/NO-GO

- На обоих текущих реальных кейсах контур визуально по мездре.
- Нет схлопывания контура.
- По времени инференс в среднем приемлем для UX (целевой ориентир до ~250 мс после оптимизации; на пилоте допускается выше).
