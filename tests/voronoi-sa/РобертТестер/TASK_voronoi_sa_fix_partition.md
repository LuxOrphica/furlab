**Индикатор обязательного гейта:** убираешь ограничение, которое держит инвариант (territory-clip держит дизъюнктность) → ВСЕГДА через гейт. Снятие ограничения = прямой риск сломать его инвариант.

**Почему:** один паттерн кусал трижды — «поднять одну метрику, молча сломать другую»: растровая метрика солвера; `gapFilled`; `core∩zone` (дал бы 98% покрытия при overlap 299 087 мм²). Прогноз по всем осям до кода ломает туннельное зрение.

**Самоконтроль (не на доверии):**
- прогноз проверяем — сказал «overlap останется 0», Роберт после фикса подтвердит/опровергнет; не сошлось → флаг;
- приёмка: улучшил одну ось, ухудшил другую — это **НЕ «готово»**. Держатся все оси одновременно. Так `core∩zone` отсекается сам: покрытие↑, overlap↑ → отклонено.

## 2. ГЛАВНЫЙ ДИАГНОЗ — дыры это дефект ПАРТИЦИИ, а не нехватка меха

Установлено по геометрии:
- `Union(ПОЛНЫЕ ядра alignedCoreContour) ∩ zone = 98.225%`
- `Union(ФРАГМЕНТЫ inZoneCoreContour) ∩ zone = 92.283%`
- разрыв **5.94 пп (22 136 мм²)** — это места, которые **накрыты хотя бы одним полным ядром, но не достались ни одному фрагменту**.

Полные ядра наезжают друг на друга на 384 170 мм². Внутри solver зона режется на **дизъюнктные территории**, каждое ядро обрезается по своей территории (`inZoneCoreContour`). Спот попадает на территорию куска, чьё **инсетнутое ядро туда не дотягивается**, а налезающее полное ядро соседа **обрезается по границе территории** → в раскрое встык дыра, хотя сырьё под ней лежит.

Подтверждение, что это партиция, а не дефицит:
- каждая внутренняя дыра (#4 — 3222 мм² в 110 мм от края; #5 — 2178 мм² в 112 мм) **на 100% накрыта полными ядрами**;
- `physicalMissingMm2` = 0 у всех кусков.

**Вывод: дыры #1–#5 закрываются БЕЗ нового материала — перераспределением территорий, а не поиском кусков.**

### Целевой инвариант партиции (проверяемый)
```
Union(inZoneCoreContour) ∩ zone  ==  Union(alignedCoreContour) ∩ zone     (с точностью до под-сетки)
```
Сейчас нарушен на 5.94 пп. Каждая точка зоны, накрытая хотя бы одним размещённым ядром, обязана достаться ровно одному фрагменту. Сейчас партиция оставляет накрытые споты ничьими.

Вероятная причина в коде: территория = ячейка по ближайшему seed'у, затем `fragment = core ∩ territory`; если ядро не заполняет территорию — дыра. Нужно, чтобы назначение спота уважало РЕАЛЬНОЕ накрытие ядром (спот идёт к ядру, которое его действительно содержит), а не к ближайшему seed'у независимо от того, дотягивается его ядро или нет.

## 3. Шаги (от якоря, по одному, с гейтом)

### Шаг 1 — вычистить врущие вторичные метрики (репорт-онли, геометрию НЕ трогаем)
В прогоне обнаружено:
- `metrics.rasterSeamArtifactMm2` = **28 579** = ровно `residualInteriorMm2`. Код объявляет ВСЮ внутреннюю недостачу «растровым шовным артефактом». **Ложь** — #4/#5 в 110 мм от края, это реальные дыры.
- `phaseA.geomCoveragePct` = 0, `phaseA.geomResidualMm2` = 0 — фантомные (нули при реальном остатке 28 750). Выпилить.
- `phaseA.bestRasterCoveragePct` = **100** — растр считает зону закрытой полностью. Возврат «лучшего по растру» бесполезен.

Сделать:
1. `rasterSeamArtifactMm2` — артефактом считать ТОЛЬКО под-сеточный **периметровый** слайвер (компонент, гибнущий при эрозии `buffer(−1.5мм)`). Любой внутренний остаток — реальная дыра, в артефакт не списывать.
2. Удалить фантомные `geomCoveragePct` / `geomResidualMm2` (или чинить, но проще удалить — они мёртвые).
3. Best-state и любой «coverage» для выхода/возврата считать **полигонально по фрагментам**: `area(Union(inZoneCoreContour) ∩ zone)/area(zone)`. Растровый `bestRasterCoveragePct` из логики выхода/возврата убрать.

**Гейт:** прогон завершается; reported coverage == coverage независимого верификатора; `resultStatus` отражает реальные дыры; геометрия (число кусков, фрагменты) не изменилась. Коммит.

### Шаг 2 — закрыть партиционные дыры перераспределением (ОСНОВНАЯ работа)
Реализовать партицию, удовлетворяющую инварианту из §2: дизъюнктные фрагменты должны покрывать **ровно** `Union(ядер)∩zone`, не меньше. Где территория-владелец не достаёт инсетнутым ядром, а сосед достаёт полным — спот отдать соседу (расширить его фрагмент), сохраняя дизъюнктность.

Ожидаемо на zone_1 без новых кусков и без перекрытий:
- покрытие 92.283% → ~98.2% (уровень `Union(ядер)`);
- внутренние дыры #4/#5 → 0; толстые провалы #1–#3 у вогнутого края — закрыться или резко уменьшиться;
- фрагменты дизъюнктны (overlap 0), дубли 0, суб-мин 0;
- остаток = только то, что **не накрыто ни одним ядром** (истинная нехватка, ~1.8%).

**Гейт:** независимый верификатор по фрагментам: покрытие выросло, внутренних дыр 0, оси 3/4 чисты, дизъюнктность держится. Коммит.

## 4. Приёмка (4 оси, ВСЕ по фрагментам `inZoneCoreContour`)

1. **Покрытие** = `Union(inZoneCoreContour)∩zone / zone` — как можно выше, без дробления.
2. **Реальные дыры** = компоненты `zone − Union(фрагментов)`, выжившие при `buffer(−1.5)`: **внутренних 0**; периметровые — только тонкие слайверы (≥55% периметра на контуре зоны). Толстый провал (выпирает в поле) недопустим, даже если касается края.
3. **Валидность** = MBR-short каждого фрагмента ≥ `minWidthMm` (по короткой грани мин. повёрнутого прямоугольника, НЕ bbox): **0 нарушителей**.
4. **Уникальность** = каждый `inventoryTag` ≤ 1 раз: **0 дублей**.
+ дизъюнктность фрагментов ≈ 0; `Union∩zone ≤ 100%`.

## 6. Что проверит независимый верификатор (shapely/pyclipper, отдельно от кода солвера)
