Core-Priority Territory
Construction (CPT)

v1.0 · 2026-06-21 · edges-first · minWidth invariant при построении

Power-Voronoi + постфильтр исчерпан: zone_4 85% разрыва — несмежные орфаны дропнутого широкого куска, связно непатчимы. zone_2 — thrash v4/v5/v6: реанимация тонкого куска добавляет исключительное ядро в covC → R2 растёт, а не падает. Новый конструктив строит territory от ядер наружу с гарантией MBR ≥ minWidth при построении, не постфильтром.

§1 — Перенесённые инварианты
INV-1 · minWidth
MBR-short(fragment_k) ≥ minWidthMm — гарантируется при построении. Суб-минимальные фрагменты не создаются вовсе: thin-регион либо поглощается валидным соседом, либо не выделяется в отдельный кусок.
INV-2 · Coverage by polygon
Принадлежность клетки куску определяется полигоном ядра (Clipper intersection area), а не 3мм растровой маской. Маска была причиной несмежных орфанов в zone_4.
INV-3 · No revival
Тонкие куски не реанимируются: они либо поглощаются (territory → BFS соседям) либо расширяются до MBR ≥ minWidth без добавления чужого core в covC.
INV-4 · Tongue + body
Fragment куска может тянуться язычком в карман (узкая часть territory). MBR-short считается по всему полигону core_k ∩ territory_k — широкое тело обеспечивает MBR ≥ 70 даже при наличии языка.
INV-5 · Determinism
Фиксированный порядок каждой фазы + явные тай-брейки (inventoryTag asc → piece index asc). Тот же seed → тот же output побайтно.
INV-6 · No silent drop
Каждый кусок поглощённый при построении логируется [CPT-ABSORB] с тегом и MBR. Никаких неявных исчезновений без NOTE.
§2 — Алгоритм по фазам
PH-0 Core Coverage Pre-computation compute · O(K·M)

Для каждого куска k и каждой клетки idx в bounding-box ядра вычисляем точный overlap через Clipper:

  1. Для k = 0..K−1: bbox(core_k) → список клеток idx в bbox.
  2. Для каждого такого idx: cov[k][idx] = area(intersect(core_k_poly, cell_rect_idx)) (Clipper, мм²).
  3. Строим обратный индекс: covByCell[idx] → [(k, score)], отсортированный по score desc.
Типичная оценка: 50 кусков × 150 клеток в bbox = 7 500 Clipper-пересечений. В текущей реализации начальный BFS уже дороже — накладные расходы приемлемы. Оптимизация: skip если area < 0.01 мм² (sub-pixel).
PH-1 Exclusive + Contested Assignment assign · O(N_cells)

Первичный раздел по покрытию ядрами — без Voronoi-центроидов, которые не знают про форму core.

  1. Для каждой клетки idx: candidates = {k : cov[k][idx] > 0}.
  2. Если |candidates| == 1 → эксклюзив: assignment[idx] = k. Клетка закреплена за единственным покрывающим куском.
  3. Если |candidates| > 1 → contested: assignment[idx] = argmax_k(cov[k][idx]). Тай-брейк: tag asc → idx asc.
  4. Если |candidates| == 0 → assignment[idx] = UNCOV (→ PH-2).
Это закрывает 85% разрыва zone_4: эксклюзивные клетки дропнутого широкого куска теперь корректно относятся к нему, а не к ближайшему центроиду, который его core не покрывает.
PH-2 Edges-First Uncovered Fill expand · edge-first BFS

Клетки без покрытия ядром (UNCOV) заполняются в порядке «от края зоны внутрь». Это гарантирует что краевые и карманные регионы закрываются намеренно, а не достаются объедкам интерьерного BFS.

  1. BFS от boundary zone → edge_dist[idx] для всех клеток зоны.
  2. Сортируем UNCOV-клетки: (edge_dist asc, row asc, col asc).
  3. Для каждой idx в этом порядке: смежные куски k (из assignment соседей).
  4. Score k: cov[k][idx] если > 0, иначе обратное расстояние до core_k (аппроксимация).
  5. assignment[idx] = argmax_k(score). Тай-брейк: tag asc → idx asc.
  6. Если нет смежных survivors → defer (orphan, → PH-5).
«Язык в карман»: краевая клетка в узком кармане получает assignment того куска, чей core имеет наибольший overlap. Fragment этого куска вытянется в карман, но MBR-short считается по всему полигону включая широкое тело → INV-4 соблюдён.
PH-3 MBR Enforcement — Expand or Absorb construction-time minWidth

После PH-1 и PH-2 каждый кусок проверяется на MBR-инвариант. Тонкие куски исправляются при построении — это ключевое отличие от постфильтра.

  1. Для k в (tag asc, idx asc): вычислить fragment_k = Clipper(core_k ∩ territory_k).
  2. Если MBR-short(fragment_k) ≥ minWidthMm → k валиден, continue.
  3. Attempt E (expand): перебрать граничные клетки k-territory (клетки соседних survivors, смежные с k). Для каждой в порядке (cov[k][idx] desc, tag asc, idx asc):
      — Временно: assignment[idx] = k.
      — Проверить donor j: MBR-short(core_j ∩ territory_j_new) ≥ minWidthMm.
      — Проверить k: MBR-short(core_k ∩ territory_k_new) ≥ minWidthMm.
      — Если оба OK → зафиксировать, пометить k и j в affectedSurvivors, break.
  4. Если Attempt E не дал MBR ≥ minWidth → Absorb: assignment[idx] = ABSORB для всех ячеек k. Убрать k из resultPlacements. Log: [CPT-ABSORB].
Важно: при Absorb ячейки k не уходят в -1 (orphan), а в ABSORB (→ PH-4 BFS). Тем самым нет несмежных orphan-зон которые были причиной zone_4 регрессии. Donor-safety check (шаг 3) предотвращает chain-reaction thin.
PH-4 Absorbed Cell BFS Redistribution BFS · coverage polygon

Ячейки поглощённых кусков (ABSORB) перераспределяются через глобальный BFS с mask-check по полигону, а не по растровой маске.

  1. Seed BFS: все ячейки валидных survivors (assignment = k, k ∈ validSet).
  2. BFS-очередь инициализируется в растровом порядке (row asc, col asc) — детерминизм.
  3. Для каждой ABSORB-ячейки ni: mask_check(k, ni) = cov[k][ni] > 0 (полигонный overlap, не растровая маска).
  4. Tie-break при конкуренции k1/k2: cov[k1][ni] vs cov[k2][ni] desc; при равенстве — tag asc, idx asc.
  5. Необслуженные ABSORB-ячейки (нет достижимого survivor) → ORPHAN (→ PH-5).
PH-5 Final Orphan Sweep BFS · fallback

Оставшиеся ORPHAN-клетки (не накрытые ни одним core, не достигнутые PH-4 BFS) присваиваются ближайшему survivor по геодезическому расстоянию (BFS по зоне). Это не увеличивает covF (core не покрывает), но заполняет территорию и предотвращает «дыры» в зоне.

covF = 100% недостижим: ядро инсетнуто на припуск → краевое кольцо шириной с припуск кроется только телом. Цель — Ось2Б ≤ допуска, не covF=100. Orphan sweep переносит physMissing в Ось2А (NOTE, не FAIL).
PH-6 Fragment Rebuild + Safety Net rebuild · safety

Перестраиваем inZoneCoreContour для всех survivors в affectedSurvivors (те кто участвовал в Attempt E как donor или recipient, и absorbed-BFS receivers). Survivors не затронутые PH-3/4 сохраняют initial fragment из PH-1 — не перестраиваются.

  1. Safety net: для каждого перестроенного fragment_k: если MBR-short < minWidthMm — NOTE + drop. По инварианту построения не должно происходить; log как [CPT-SAFETY-DROP] если произошло.
  2. Финальная проверка: nPolyOuter(territory_k) == 1 (связность). Если нет — взять наибольшую компоненту, остаток → ORPHAN.
